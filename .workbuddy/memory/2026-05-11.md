## 内网离线构建方案——依赖包打包进 git

### 需求
公司内网无法访问外部 PyPI/npm registry，需要将依赖包预先打包放入 git，构建时不访问外网。

### 方案设计
将所有外部依赖（Python pip 包 + Node npm 包）打包存入项目 `vendor/` 目录并提交 git，离线 Dockerfile 从本地安装。

### 新增文件

| 文件 | 说明 |
|------|------|
| `backend/Dockerfile.offline` | 后端离线构建 Dockerfile（`pip --no-index --find-links=./vendor`）|
| `frontend/Dockerfile.offline` | 前端离线构建 Dockerfile（`npm cache add` + `npm ci --prefer-offline`）|
| `scripts/pack-backend-deps.sh/.ps1` | 后端 Python 包下载脚本（pip download）|
| `scripts/pack-frontend-deps.sh/.ps1` | 前端 Node 包打包脚本（npm pack from node_modules）|
| `pack-offline-deps.sh/.ps1` | 一键打包（后端+前端，可选 git commit）|
| `docs/OFFLINE_BUILD_GUIDE.md` | 完整使用文档 |

### 关键技术点
- **Python**：`pip download --platform manylinux2014_x86_64 --python-version 3.10` 下载与容器平台匹配的 whl，Dockerfile 用 `--no-index --find-links=./vendor` 安装
- **Node**：从 `node_modules/` 用 `npm pack` 生成 `.tgz`，Dockerfile 用 `npm cache add` 加载后 `npm ci --prefer-offline` 安装
- `.gitignore` 增加 `!backend/vendor/` 和 `!frontend/vendor/` 例外规则，确保 vendor 目录被 git 追踪

### 工作流
```
有网机器：./pack-offline-deps.ps1 → git push
内网服务器：git pull → docker-compose up -d --build
```
