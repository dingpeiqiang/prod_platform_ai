## 推荐引擎 FormInstance.form_code bug 修复

**改动文件**：

## 聊天动态更新表单功能（增量更新）

**需求**：用户说"请假3天"、"改成事假"时，直接更新右侧面板的表单字段，无需重新生成表单。

**改动文件**：

1. **后端：意图识别 prompt 增强**
   - `config/prompts/smart_intent_recognition.txt`
   - 新增 `intentType: "form_update"` 意图类型定义
   - 说明区分"首次生成表单"和"更新现有表单字段"

2. **后端：chat.py SSE 返回 intentType**
   - 新增 `form_update` 意图处理分支（第 728 行后）
   - done 事件返回 `intentType: "form_update"` 和 `intentData`

3. **前端：ChatAssistant.vue**
   - SSE 解析时提取 `intentType` 字段
   - `sendMessage` 中根据 `intentType` 区分处理：
     - `form`：调用 `generateForm` 生成新表单
     - `form_update`：调用 `updateFormFields` 增量更新
   - 新增 `updateFormFields()` 方法：检查现有表单，提取 `extractedFields` 更新字段值

4. **前端：DynamicForm.vue**
   - watch 增强：优先使用 `field.value` 属性更新表单数据
   - 支持外部通过修改 schema.fields[].value 触发表单更新

**交互流程**：
```
用户: 帮我请假      → intentType="form"   → 生成请假表单
用户: 请假3天       → intentType="form_update" → 更新 end_date 字段
用户: 改成事假      → intentType="form_update" → 更新 leave_type 字段
```
