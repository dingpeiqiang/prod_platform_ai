# 2026-04-29 工作日志

## 意图处理器架构改造完成

### 问题
chat.py 的 chat_stream() 函数中有 436 行 if/elif 硬编码链（6种意图），新增意图需同时修改3处无解耦

### 改造方案
引入 IntentHandler 注册器模式，参考 MCPToolHub 的 @mcptool 装饰器模式

### 后端改造
- 新增 `backend/app/intent/` 模块（8个文件，共796行）：
  - `base.py`（61行）：BaseIntentHandler ABC + IntentContext dataclass
  - `registry.py`（120行）：IntentHandlerRegistry 单例 + @intent_handler 装饰器
  - `utils.py`（213行）：SSE工具函数(sse/thinking/reasoning) + 推荐合并 + 流式聊天
  - `handlers/`：6个独立处理器文件（form/form_update/delete_form/manage_history/configure/chat）
- 修改 `chat.py`：437行 if/elif 链替换为 `registry.dispatch(intent_type, ctx)` 一行分发
- chat.py 从 ~1970 行降至 1534 行
- 向后兼容：SSE 事件协议完全不变，其余 20+ 端点零影响

### 前端改造
- 新增 `frontend/src/components/intent-panels/`（5个文件，共294行）：
  - `intent-registry.js`（82行）：SSE事件处理器 + 意图后处理器注册器
  - `DeleteResultPanel.vue`（65行）：删除结果面板
  - `HistoryPanel.vue`（119行）：历史数据维护面板
  - `ChatPanel.vue`（14行）/ `FormIntentPanel.vue`（14行）：占位面板
- 修改 `ChatAssistant.vue`：
  - handleEvent switch/case 中 config/delete_form/manage_history 3个分支改为注册器分发
  - 模板中 ~130 行内联 HTML 替换为 DeleteResultPanel + HistoryPanel 组件
  - 意图后处理 if/elif 链改为注册器查表
  - 从 ~2049 行降至 1961 行

### 新增意图的标准流程
1. 新建 `handlers/xxx_handler.py`，继承 BaseIntentHandler
2. 在 `__init__.py` 添加 import + register
3. 前端新建面板组件 + 注册事件处理器
4. **无需修改 chat_stream() 或 handleEvent 主逻辑**
