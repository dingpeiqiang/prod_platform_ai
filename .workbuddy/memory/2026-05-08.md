## 未完成表单拦截功能

### 功能描述
当表单状态为未完成时（如正在填写或待确认提交），用户尝试发起新的表单任务会被拦截，提示用户先完成或关闭当前表单。

### 实现位置
- `frontend/src/components/ChatAssistant.vue`

### 修改内容

1. **`form` 意图后处理器拦截逻辑** (line 348-373)
   - 检测条件：`currentFormSchema` 存在且 `currentFormSubmitted` 为 false，或 `pendingConfirmForm` 存在
   - 拦截后显示提示消息，提供"完成/提交"和"取消"两种处理方式

2. **`doSendMessage` 新增表单操作命令处理** (line 958-976)
   - 检测"完成/提交"关键词 → 调用 `handleConfirmSubmitForActiveForm()` 引导用户完成表单
   - 检测"取消/算了/不要了"关键词 → 调用 `handleFormCancel()` 关闭表单

3. **新增 `handleConfirmSubmitForActiveForm` 函数** (line 1887+)
   - 显示引导消息，提示用户在右侧面板完成表单

4. **修复 `handleFormCancel`** (line 1867)
   - 添加 `currentFormSubmitted.value = false` 重置提交状态
