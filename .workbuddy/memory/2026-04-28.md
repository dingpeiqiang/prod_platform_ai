## 修复 MiniMax M2.7 reasoning-only 响应问题
- 问题：MiniMax M2.7 有时把正式回答放在 reasoning 字段里，content 为空
- 症状：`💭 Reasoning (4480 chars)` + `LLM 返回为空，耗时 15.7s`
- 修复1：5 个 LLM 调用方法修改 fallback 逻辑：
  - reasoning 中有 JSON → 提取 JSON 作为 content
  - reasoning 中无 JSON → 返回 None（不再返回 reasoning 原文，避免 JSON 解析必然失败）
  - 影响方法：_call_custom_with_reasoning, _call_custom_stream_sync, _call_custom, _call_ollama_with_reasoning, _call_ollama
- 修复2：smart_intent_recognition.txt prompt 末尾加强 JSON 输出约束
  - 要求 JSON 直接输出，不用代码块包裹
  - 针对支持 reasoning 的模型，要求 JSON 放在 content 中
- 修复3：chat.py 意图识别增加重试逻辑
  - 当 content 为空但 reasoning 有内容时，用简化 prompt 重试一次
  - 重试 prompt 追加格式强调："请直接输出 JSON，不要在 JSON 之外输出任何分析文本"
  - 重试仍失败则走降级流程

## 修复 chat 意图下 thinking/reasoning 内容泄露到前端
- 问题：用户问"你能做什么"，回复中混入模型内部思考过程（💭 逐 token 碎片），显示异常

## 修复 UnboundLocalError: intent_type referenced before assignment
- 问题：LLM 调用异常时（或返回空），`intent_type` 未赋值就被第 640 行引用
- 错误：`UnboundLocalError: local variable 'intent_type' referenced before assignment`
- 根因：`intent_reasoning and intent_type in (...)` 判断在 `intent_type = intent_data.get(...)` 赋值之前执行
- 修复：将 `_reasoning()` 发送逻辑从 intent_type 赋值前移到赋值后（JSON 解析成功、intent_type 已有值之后）
- 附加效果：如果 JSON 解析失败（异常路径），reasoning 本来就不该发，逻辑更合理

## 历史数据导入器 v3（JSONL原生，支持 entities 层级结构）

### 背景
- v1: DB连接+SQL查询（用户不想配数据库）
- v2: CSV扁平格式（无法支持复杂表单的嵌套/层级关系）
- **v3**: JSONL为核心，每行一个完整JSON对象

### 核心改动
1. **import_history.py 重写**:
   - 数据读取：优先 JSONL（每行一个 JSON 对象），兼容旧 CSV
   - `flatten_record()`: 递归将嵌套 JSON 拍平为一层键值对（用于 FormHistory 频次统计）
     - dict → `parent.child` 展开
     - list → `items.0.field` 数组下标展开
     - 标量列表 → `, ` 拼接
   - FormInstance.data 保存**原始完整 JSON**（保留层级结构）
   - FormHistory 使用**拍平后的字段**（用于推荐引擎查询）
   - `_extract_timestamp()`: 自动从记录中提取时间戳（保留源系统提交时间）
   - dry-run 报告增强：区分顶层字段和嵌套子字段分布，显示首条记录样例

2. **目录结构**（每个表单类型 2 个文件）:
   ```
   config/import_data/
     ├── {formCode}.schema.json    可选：值转换规则 + userField
     └── {formCode}.data.jsonl     必填：每行一个 JSON 对象（支持任意嵌套）
   ```

3. **已创建示例数据**:
   - leave.data.jsonl: 12 条请假记录（扁平）
   - sales_order.data.jsonl: 10 条销售订单（扁平）
   - tariff_filing_publicity.data.jsonl: 12 条资费备案（30 字段复杂表单）
   - 对应的 schema.json（含 valueTransform 规则）

### 关键能力
| 能力 | 说明 |
|------|------|
| 嵌套支持 | dict/array 任意深度，自动拍平写入 FormHierarchy |
| 时间戳保留 | 从源数据提取 submitted_at/order_date 等，推荐引擎时间衰减准确 |
| 注释支持 | # 开头的行自动跳过 |
| 值转换 | schema.json valueTransform 配置第三方编码→标准值映射 |
| dry-run | 预览含嵌套结构检测、字段分布、首条样例 |
