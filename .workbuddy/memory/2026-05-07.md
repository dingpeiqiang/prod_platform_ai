## 聊天记录数据库持久化
- 前端新建 `src/services/chatApi.js`：封装 createSession/saveMessage/loadMessages/getSessions/deleteSession/updateSessionTitle
- 后端新增 `POST /api/v1/chat/sessions/{session_id}/messages` 端点，保存单条消息到 chat_messages 表
- ChatAssistant.vue：叠加 DB 持久化
  - 发送消息后 `await saveMessage()` 落库
  - AI 回复结束后 `saveMessage()` 落库
  - mount 时 `apiLoadMessages()` 从库加载
  - `watch(dbSessionId)` 监听会话切换
  - 首次发消息时 `ensureDbSession()` 创建 DB 会话
- App.vue：管理数据库会话生命周期
  - 登录后 `initDbSessions()` 同步本地会话与 DB
  - 通过 `@session-init` 回调回填本地会话的 `dbSessionId`
  - 删除会话时 `apiDeleteSession()` 同步删除 DB 记录
- 会话数据结构：每条本地会话增加 `dbSessionId` 字段映射到数据库

## 修复：历史会话加载逻辑（第二次）
- ChatAssistant `watch(() => props.sessionId)` 切换会话时增加 DB 加载分支：有 `currentDbSessionId` 则 `apiLoadMessages()`，无则降级 localStorage（之前只走 localStorage）
- ChatAssistant `onMounted` 移除错误的 `dbMsgs.length === 0 → loadMessages()` 回退：有 `dbSessionId` 时永远以 DB 为准，DB 空则显示空会话（旧版无 `dbSessionId` 的会话才走 localStorage）
- 清理无用的 `dbSessionLinked` 状态变量
