## 🏗️ 架构变化

### 之前

```
用户输入 → AgentExecutor → MCP 工具（不含工作流）
                              ↓
                         ❌ 无法调用工作流
```

### 现在

```
用户输入 → AgentExecutor → MCP ToolHub
                              ├─ form_generate
                              ├─ kb_qa
                              ├─ llm_chat
                              └─ execute_workflow ← 新增！
                                   ↓
                            WorkflowExecutor
                                   ↓
                            执行可视化工作流
```

## 💡 使用场景示例

### 场景 1: 直接执行工作流

**用户**: "帮我处理订单 ORD-12345"

**LLM 决策**:
```json
{
  "tool_name": "execute_workflow",
  "arguments": {
    "workflow_code": "order_processing",
    "inputs": {"order_id": "ORD-12345"}
  }
}
```

### 场景 2: 先查询再执行

**用户**: "我需要一个审批流程"

**LLM 步骤**:
1. 调用 `list_workflows(category="approval")`
2. 分析返回的工作流列表
3. 选择合适的工作流
4. 调用 `execute_workflow` 执行

### 场景 3: 查看工作流详情

**用户**: "订单处理工作流需要哪些参数？"

**LLM 决策**:
```json
{
  "tool_name": "get_workflow_detail",
  "arguments": {
    "workflow_code": "order_processing"
  }
}
```
