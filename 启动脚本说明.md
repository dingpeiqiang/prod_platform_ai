# AI驱动动态表单 - 启动脚本使用说明

## 🚀 快速启动

### 方式一：一键启动（推荐）
直接双击 `start-all.bat`，会自动启动后端和前端服务。

### 方式二：分别启动
1. 先启动后端：双击 `start-backend.bat`
2. 再启动前端：双击 `start-frontend.bat`

---

## 🔧 脚本功能

### start-backend.bat
- ✅ 自动检查Python环境
- ✅ 自动创建/激活虚拟环境
- ✅ 自动检查端口占用（8000/8001）
- ✅ 支持生产/开发模式选择
- ✅ 日志输出到终端和文件

### start-frontend.bat  
- ✅ 自动检查Node.js环境
- ✅ 自动安装npm依赖
- ✅ 自动检测后端服务端口
- ✅ 动态配置API代理地址
- ✅ 自动打开浏览器

### start-all.bat
- ✅ 一键启动前后端
- ✅ 自动检测端口占用
- ✅ 显示服务地址信息

---

## 📊 服务地址

启动后服务地址：

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端界面 | http://localhost:5173 | 浏览器访问 |
| 后端API | http://localhost:8000 或 8001 | 自动检测 |
| API文档 | http://localhost:8000/docs | Swagger文档 |

## ⚠️ 常见问题

### 1. 端口被占用
如果8000端口被占用，脚本会自动切换到8001端口。

手动解决：
```bash
# 查找占用端口的进程
netstat -ano | findstr :8000

# 结束进程
taskkill /F /PID <进程ID>
```

### 2. 前端无法连接后端
检查后端是否正常运行，确保API代理端口一致。

### 3. 依赖安装失败
```bash
# 手动安装后端依赖
cd backend
pip install -r requirements.txt

# 手动安装前端依赖
cd frontend
npm install
```
