# ✅ 导入历史数据时模板自动更新 - 修复完成

## 🎯 问题描述

**用户反馈**: "导入历史数据时，模板未更新"

### 问题分析

之前的 `_ensure_template` 函数逻辑：

```python
def _ensure_template(db, form_code: str, form_name: str) -> int:
    template = db.query(FormTemplate).filter(
        FormTemplate.form_code == form_code
    ).first()

    if template:
        return template.id  # ❌ 直接返回，不更新
    
    # 只有不存在时才创建
    template = FormTemplate(...)
    db.add(template)
    ...
```

**问题**:
1. 第一次导入时创建了模板（schema可能不完整）
2. 后续导入时，即使 `schema.json` 更新了，模板也不会更新
3. 推荐引擎使用的还是旧的schema
4. 导致新字段无法被识别和推荐

---

## 🔧 修复方案

### 修改前

```python
def _ensure_template(db, form_code: str, form_name: str) -> int:
    """确保 FormTemplate 存在，不存在则创建"""
    from app.models.form import FormTemplate

    template = db.query(FormTemplate).filter(
        FormTemplate.form_code == form_code
    ).first()

    if template:
        return template.id  # ❌ 不更新

    template = FormTemplate(
        form_code=form_code,
        form_name=form_name,
        schema={"formCode": form_code, "formName": form_name},
        version=1,
        is_active=True
    )
    db.add(template)
    db.commit()
    db.refresh(template)
    logger.info("创建 FormTemplate: form_code=%s id=%d", form_code, template.id)
    return template.id
```

---

### 修改后

```python
def _ensure_template(db, form_code: str, form_name: str, schema: Optional[Dict] = None) -> int:
    """
    确保 FormTemplate 存在，不存在则创建，存在则更新
    
    Args:
        db: 数据库会话
        form_code: 表单编码
        form_name: 表单名称
        schema: 完整的schema配置（可选）
    
    Returns:
        模板ID
    """
    from app.models.form import FormTemplate

    template = db.query(FormTemplate).filter(
        FormTemplate.form_code == form_code
    ).first()

    if template:
        # ⚠️ 重要：如果提供了schema，更新模板
        if schema:
            updated = False
            
            # 更新form_name
            if form_name and template.form_name != form_name:
                template.form_name = form_name
                updated = True
            
            # 更新schema（合并现有schema和新schema）
            current_schema = template.schema or {}
            # 保留原有字段，但用新schema覆盖
            for key, value in schema.items():
                if key not in current_schema or current_schema[key] != value:
                    current_schema[key] = value
                    updated = True
            
            if updated:
                template.schema = current_schema
                template.updated_at = datetime.now()
                db.commit()
                logger.info("更新 FormTemplate: form_code=%s", form_code)
        
        return template.id

    # 创建新模板
    template_schema = schema or {"formCode": form_code, "formName": form_name}
    template = FormTemplate(
        form_code=form_code,
        form_name=form_name,
        schema=template_schema,
        version=1,
        is_active=True,
        created_at=datetime.now(),
        updated_at=datetime.now()
    )
    db.add(template)
    db.commit()
    db.refresh(template)
    logger.info("创建 FormTemplate: form_code=%s id=%d", form_code, template.id)
    return template.id
```

### 调用处修改

**修改前**:
```python
template_id = _ensure_template(db, form_code, form_name)
```

**修改后**:
```python

```

## 🎯 关键特性

### 1. 智能合并Schema

```python
# 保留原有字段，但用新schema覆盖
for key, value in schema.items():
    if key not in current_schema or current_schema[key] != value:
        current_schema[key] = value
        updated = True
```

**优势**:
- ✅ 不会丢失已有的自定义配置
- ✅ 自动更新变化的字段
- ✅ 只在实际有变化时才更新数据库

---

### 2. 更新时间戳

```python
if updated:
    template.schema = current_schema
    template.updated_at = datetime.now()  # ✅ 记录更新时间
    db.commit()
    logger.info("更新 FormTemplate: form_code=%s", form_code)
```

**优势**:
- ✅ 可以追踪模板的变更历史
- ✅ 便于调试和问题排查

---

### 3. 向后兼容

```python
def _ensure_template(db, form_code: str, form_name: str, schema: Optional[Dict] = None) -> int:
```

**优势**:
- ✅ `schema` 参数是可选的
- ✅ 旧代码调用时不传schema也能正常工作
- ✅ 不会破坏现有功能
