# 流式输出功能说明

## 概述

聊天接口现已支持**真正的流式输出**，用户可以在聊天窗口中看到实时的推理过程和回复内容（打字机效果）。

## 功能特性

### 1. **实时推理过程显示**

用户在聊天窗口会看到以下推理步骤：

```
🔍 正在分析用户意图...
🧠 调用 LLM 进行意图识别...
✅ 意图识别完成: form/chat
📋 识别到表单类型: leave
📝 提取到 3 个字段
💬 正在生成回复...
🤖 调用 LLM 生成回复...
```

### 2. **流式文本输出**

- **聊天回复**：逐字显示，类似打字机效果
- **表单意图**：显示完整的 JSON 结果
- **Skills Fallback**：同样支持流式输出

### 3. **事件类型**

后端通过 Server-Sent Events (SSE) 发送以下事件类型：

| 事件类型 | 说明 | 示例 |
|---------|------|------|
| `thinking` | 推理过程消息 | "🔍 正在分析用户意图..." |
| `text_start` | 文本开始标志 | - |
| `text` | 文本片段 | "你好" |
| `text_end` | 文本结束标志 | - |
| `result` | 结构化结果（JSON） | 意图识别结果 |
| `done` | 完成标志 | `{"isForm": true}` |
| `error` | 错误信息 | - |

## 技术实现

### 后端实现

**文件**: `backend/app/api/chat.py`

#### 核心流程

1. **接收请求**：获取用户消息和历史记录
2. **意图识别**：调用 LLM 识别用户意图（form/chat）
3. **分支处理**：
   - **表单意图**：返回结构化数据，前端生成表单
   - **聊天意图**：流式输出 LLM 生成的回复
4. **Fallback**：如果 LLM 失败，使用 Skills 处理

#### 关键代码

```python
# 流式生成器
async def stream_generator():
    # 1. 显示推理步骤
    yield f"data: {json.dumps({'type': 'thinking', 'content': '🔍 ...'})}\n\n"
    
    # 2. 调用 LLM
    intent_result = llm_service._call_llm_sync(intent_prompt)
    
    # 3. 流式输出回复
    for i in range(0, len(reply_text), 2):
        chunk = reply_text[i:i+2]
        yield f"data: {json.dumps({'type': 'text', 'content': chunk})}\n\n"
        await asyncio.sleep(0.03)  # 控制输出速度
    
    # 4. 完成标志
    yield f"data: {json.dumps({'type': 'done', 'isForm': False})}\n\n"
```

### 前端集成

前端需要使用 EventSource 或 fetch API 接收 SSE 流：

```javascript
const response = await fetch('/api/v1/chat/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ messages: [...] })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  const lines = chunk.split('\n\n');
  
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = JSON.parse(line.slice(6));
      
      switch (data.type) {
        case 'thinking':
          // 显示推理过程
          showThinking(data.content);
          break;
        case 'text':
          // 追加文本
          appendText(data.content);
          break;
        case 'done':
          // 完成
          if (data.isForm) {
            generateForm();
          }
          break;
      }
    }
  }
}
```

## 配置参数

### 流式输出速度控制

在 `chat.py` 中可以调整：

```python
# 推理步骤之间的延迟
await asyncio.sleep(0.2)  # 默认 200ms

# 文本输出速度
await asyncio.sleep(0.03)  # 每 2 个字符延迟 30ms
```

**建议值**：
- 推理步骤：0.2-0.5 秒（让用户看清每个步骤）
- 文本输出：0.02-0.05 秒（流畅的打字机效果）

## 性能优化

### 1. 减少延迟

如果希望更快的响应，可以减少 `asyncio.sleep` 的时间：

```python
await asyncio.sleep(0.01)  # 更快的输出
```

### 2. 批量输出

增加每次输出的字符数：

```python
for i in range(0, len(reply_text), 5):  # 每次 5 个字符
    chunk = reply_text[i:i+5]
    ...
```

### 3. 缓存推理步骤

对于常见问题，可以缓存推理步骤，避免重复调用 LLM。

## 故障排查

### 问题 1：看不到流式输出

**可能原因**：
- 前端未正确解析 SSE 格式
- 浏览器缓冲了响应

**解决方案**：
- 检查响应头是否包含 `X-Accel-Buffering: no`
- 确保前端使用正确的解析逻辑

### 问题 2：输出太快/太慢

**解决方案**：
调整 `asyncio.sleep` 的参数：
```python
await asyncio.sleep(0.05)  # 更慢
await asyncio.sleep(0.01)  # 更快
```

### 问题 3：推理步骤不显示

**可能原因**：
- 前端未处理 `thinking` 类型的事件

**解决方案**：
检查前端的 event handler 是否正确处理所有事件类型。

## 与 Ollama 集成的关系

当前实现中：
- ✅ **意图识别**：调用 Ollama（非流式，因为需要解析 JSON）
- ✅ **聊天回复**：调用 Ollama（非流式，然后逐字输出模拟流式）
- ⚠️ **真正的 LLM 流式**：Ollama 支持流式 API，但需要额外实现

### 未来改进：真正的 LLM 流式

Ollama 支持流式 API（`stream: true`），可以实现真正的 LLM token 级流式输出：

```python
# 伪代码
response = requests.post(url, json={
    "model": model,
    "messages": messages,
    "stream": True  # 启用流式
}, stream=True)

for line in response.iter_lines():
    if line:
        data = json.loads(line)
        token = data.get('message', {}).get('content', '')
        yield f"data: {{'type': 'text', 'content': '{token}'}}\n\n"
```

这将是下一步优化的方向。
