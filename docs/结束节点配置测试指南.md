# 结束节点配置功能 - 快速测试指南

## 测试步骤

### 1. 启动开发环境

```bash
# 前端
cd frontend
npm run dev

# 后端(可选,如果需要保存工作流)
cd backend
python -m uvicorn app.main:app --reload
```

### 2. 打开工作流编辑器

访问: `http://localhost:5173/workflow-editor`

### 3. 创建测试工作流

1. 从左侧面板拖拽以下节点到画布:
   - 开始节点
   - LLM节点
   - 结束节点

2. 连接节点: 开始 → LLM → 结束

### 4. 测试结束节点配置

#### 4.1 打开配置面板
- **方法1**: 双击结束节点
- **方法2**: 选中结束节点,按 `Ctrl+G`

#### 4.2 验证界面元素

检查以下区域是否正确显示:

✅ **顶部标题栏**
- [ ] 左侧显示"编辑工作流"(黑色加粗)
- [ ] 右侧有关闭按钮(×)

✅ **节点标识区**
- [ ] 紫色方形图标(内含白色旗帜)
- [ ] 输入框显示"结束节点"
- [ ] 下方有说明文字

✅ **选择回答模式**
- [ ] 标签显示"选择回答模式"
- [ ] 下拉框包含: 文本/JSON/Markdown/HTML
- [ ] 默认选中"文本"

✅ **输出配置区**
- [ ] 标题"输出"带向下箭头
- [ ] 有帮助图标(i)
- [ ] 有蓝色 + 号按钮
- [ ] 点击 + 号能添加参数行
- [ ] 每行包含: 参数名输入框、来源类型下拉、值选择/输入、删除按钮

✅ **是否流式输出**
- [ ] 标签显示"是否流式输出"
- [ ] 两个单选按钮: 是/否
- [ ] 默认选中"否"

✅ **回答内容配置区**
- [ ] 标题"回答内容"带向下箭头
- [ ] 有帮助图标(i)
- [ ] "是否对话历史"开关(默认关闭)
- [ ] 多行文本框
- [ ] 占位符提示使用 {变量名} 语法

#### 4.3 测试交互功能

**测试1: 修改节点名称**
1. 在节点标识区的输入框中输入新名称,如"最终输出"
2. 关闭配置面板
3. 画布上的节点名称应更新

**测试2: 切换回答模式**
1. 在"选择回答模式"下拉框中选择"JSON"
2. 关闭配置面板
3. 重新打开,确认选择已保存

**测试3: 添加输出参数**
1. 点击输出区域的 + 号按钮
2. 填写参数名: `result`
3. 选择来源类型: "引用"
4. 应该能看到可用变量列表(如果前面有LLM节点)
5. 选择一个变量
6. 再次点击 + 号,添加第二个参数
7. 点击 × 号删除其中一个参数

**测试4: 切换流式输出**
1. 选择"是"选项
2. 关闭面板
3. 重新打开,确认选择已保存

**测试5: 编写回答内容**
1. 展开"回答内容"区域
2. 在文本框中输入: `处理结果是: {result}`
3. 开启"是否对话历史"开关
4. 关闭面板
5. 重新打开,确认内容已保存

**测试6: 折叠/展开面板**
1. 点击"输出"区域的标题
2. 确认区域可以折叠和展开
3. 对"回答内容"区域执行相同操作

**测试7: 变量引用**
1. 确保工作流中有LLM节点在结束节点之前
2. 在输出参数中,选择来源类型为"引用"
3. 下拉框应显示可用的变量
4. 格式应为: `{节点名称} (输出)`

### 5. 测试数据持久化

1. 配置所有参数
2. 点击工具栏的"保存"按钮(Ctrl+S)
3. 刷新页面
4. 重新打开工作流
5. 打开结束节点配置
6. 确认所有配置都已恢复

### 6. 测试边界情况

**测试1: 空参数列表**
- 删除所有输出参数
- 应显示提示:"暂无输出参数,点击右上角 + 号添加"

**测试2: 长文本输入**
- 在回答内容中输入很长的文本
- 文本框应支持滚动
- 可以垂直拖动调整大小

**测试3: 特殊字符**
- 在参数名中输入特殊字符
- 系统应能正确处理(建议后续添加验证)

**测试4: 无前置节点**
- 创建工作流只有结束节点
- 打开配置,变量引用下拉框应为空或禁用

### 7. 视觉检查

检查以下内容:

- [ ] 紫色渐变背景是否正确显示
- [ ] 图标是否清晰可见
- [ ] 按钮悬停效果是否正常
- [ ] 输入框聚焦时是否有高亮边框
- [ ] 单选按钮样式是否正确
- [ ] 开关组件动画是否流畅
- [ ] 折叠动画是否平滑
- [ ] 整体布局是否整齐美观

### 8. 响应式测试

1. 调整浏览器窗口大小
2. 检查配置面板在不同宽度下的表现
3. 确认没有布局错乱或内容溢出

## 预期结果

所有测试项都应该通过,界面应该:
- ✅ 美观整洁
- ✅ 交互流畅
- ✅ 功能完整
- ✅ 数据持久化正常
- ✅ 无控制台错误

## 常见问题

### Q1: 变量引用下拉框为空?
**A**: 确保结束节点之前有其他节点(如LLM、HTTP等),并且已正确连接。

### Q2: 配置没有保存?
**A**: 检查是否点击了保存按钮,或查看浏览器控制台是否有错误。

### Q3: 样式显示不正确?
**A**: 清除浏览器缓存,或硬刷新(Ctrl+F5)。

### Q4: 折叠按钮不工作?
**A**: 检查浏览器控制台是否有JavaScript错误。
