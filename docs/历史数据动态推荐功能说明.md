# 历史数据动态推荐功能说明

## 功能概述

推荐数据现在支持**基于历史表单提交数据动态生成**，同时保持静态配置作为兜底。

---

## 核心特性

### 1. 混合推荐策略
- **优先数据库历史**：从已提交的表单数据中提取推荐值
- **静态配置兜底**：当无历史数据时，使用 `config/templates/recommendations.json` 中的静态推荐
- **去重合并**：数据库推荐和静态推荐合并并去重

### 2. 智能权重计算

推荐值的最终评分由三个因素加权计算：

| 因素 | 权重 | 说明 |
|------|------|------|
| **频次评分** | 40% | 使用频次越高，评分越高（最高1分，每出现1次加0.1分） |
| **用户专属评分** | 40% | 用户自己提交过的值评分更高（最高1分，每出现1次加0.2分） |
| **时间衰减评分** | 20% | 最近使用的值评分更高（30天内线性衰减） |

**计算公式：**
```
最终评分 = (频次评分 * 0.4) + (用户专属评分 * 0.4) + (时间衰减评分 * 0.2)
```

### 3. 自动历史记录

表单提交时自动记录每个字段的历史值到 `FormHistory` 表中，包括：
- 表单实例ID
- 字段编码
- 字段值
- 用户ID（可选）
- 记录时间

## 实现细节

### 核心文件修改

| 文件 | 说明 |
|------|------|
| `app/services/history_service.py` | 重构，支持数据库+静态混合推荐 |
| `app/services/form_service.py` | 添加db参数传递 |
| `app/api/form.py` | 更新API路由，传递db会话，提交时保存历史 |

### 推荐流程

```
用户请求推荐
    ↓
读取数据库历史（最近1000条已提交表单）
    ↓
计算每个值的评分（频次+用户+时间）
    ↓
合并静态配置（去重）
    ↓
按评分降序排序
    ↓
返回Top N推荐
```

## 使用示例

### 场景1：首次使用（无历史数据）
```
用户输入：帮我填一个销售订单
推荐值来源：静态配置 (recommendations.json)
推荐结果：["张三", "李四", "王五", ...]
```

### 场景2：已有历史数据
```
用户：张三提交过3次"张三"作为客户姓名
用户：李四提交过1次"张三"作为客户姓名
用户最近提交：2天前

推荐值评分计算：
- "张三"：频次(3)0.3分 + 用户专属(3)0.6分 + 时间1分 = 0.3*0.4 + 0.6*0.4 + 1*0.2 = 0.56分
- ...

推荐结果：按评分排序
```

## 配置说明

### 静态推荐配置（兜底）
路径：`config/templates/recommendations.json`

```json
{
  "recommendations": {
    "sales_order": {
      "customer_name": ["张三", "李四", "王五"],
      "customer_phone": ["13800138000"]
    }
  }
}
```

### 可调整的参数（代码中）
在 `HistoryService._calculate_score` 方法中可调整：
- 频次权重（当前0.4）
- 用户专属权重（当前0.4）
- 时间衰减权重（当前0.2）
- 时间衰减周期（当前30天）
- 查询历史记录条数（当前1000条）

## 数据库表结构

### FormHistory 表
```sql
CREATE TABLE form_histories (
    id INTEGER PRIMARY KEY,
    form_instance_id INTEGER,
    field_code VARCHAR(255),
    field_value TEXT,
    user_id VARCHAR(100),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 扩展建议

1. **增加缓存机制**：历史推荐结果可以缓存一段时间（如1小时）
2. **支持场景过滤**：同一字段在不同表单中可以有不同的推荐策略
3. **支持黑名单**：可以配置某些值不被推荐
4. **支持人工置顶**：可以配置某些值永远排在推荐列表前面
