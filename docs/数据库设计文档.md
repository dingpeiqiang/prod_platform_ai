# 数据库设计文档

> 文档版本：v2.0
> 更新日期：2026-05-07
> 说明：聊天记录架构从紧耦合业务字段迁移为 metadata 解耦设计

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **核心表字段永不膨胀** | `chat_messages` 只存储最朴素的聊天字段，业务扩展通过 metadata 表插入 |
| **KV 扩展表** | `chat_message_metadata` 以 key-value 行存储业务字段，新增业务字段无需改表结构 |
| **避免关键字冲突** | 所有列名经过审查，不使用 SQL/MySQL 保留关键字 |
| **级联删除** | 删除会话自动清理关联消息和 metadata，由数据库外键 ON DELETE CASCADE 保证 |

## 三、表结构

### 3.1 chat_sessions（聊天会话表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | INTEGER | PK, AUTO_INCREMENT | 自增主键 |
| `session_id` | VARCHAR(64) | NOT NULL, UNIQUE | 会话唯一标识（UUID） |
| `user_id` | VARCHAR(100) | — | 用户ID（可空，支持匿名会话） |
| `title` | VARCHAR(200) | — | 会话标题（可由首条消息摘要生成） |
| `context_tags` | JSON | — | 会话标签，`["sales", "urgent"]` 等 |
| `session_metadata` | JSON | — | 会话扩展信息，`{source: "web", channel: "api"}` |
| `status` | VARCHAR(20) | DEFAULT 'active' | 会话状态，`active` / `archived` |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | — | 最后消息时间（列表排序用） |

### 3.2 chat_messages（聊天消息表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | INTEGER | PK, AUTO_INCREMENT | 自增主键 |
| `message_id` | VARCHAR(64) | NOT NULL, UNIQUE | 消息唯一标识（UUID） |
| `session_id` | VARCHAR(64) | NOT NULL, FK | 所属会话ID |
| `role` | VARCHAR(20) | NOT NULL | 角色，`user` / `assistant` / `system` |
| `content` | TEXT | NOT NULL | 消息正文（支持 Markdown / 富文本） |
| `content_type` | VARCHAR(20) | DEFAULT 'text' | 内容类型，`text` / `markdown` / `json` / `form` |
| `parent_id` | VARCHAR(64) | — | 父消息ID（多轮对话树状结构） |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 3.3 chat_message_metadata（消息 KV 扩展表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | INTEGER | PK, AUTO_INCREMENT | 自增主键 |
| `message_id` | VARCHAR(64) | NOT NULL, FK | 所属消息ID |
| `meta_key` | VARCHAR(100) | NOT NULL | 扩展字段名 |
| `value` | TEXT | — | 扩展字段值（JSON 字串或文本） |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**联合唯一索引**：`UNIQUE(message_id, meta_key)` — 同消息内 key 不可重复

## 四、metadata 字段约定

业务扩展字段通过 `chat_message_metadata` 表存储，无需修改 `chat_messages` 主表结构。

### 常用字段

| meta_key | value 示例 | 说明 |
|----------|-----------|------|
| `intent_type` | `"form"` | 意图类型：`form` / `chat` / `configure` / `delete_form` 等 |
| `form_code` | `"leave"` | 表单编码 |
| `extracted_fields` | `"{\"leave_type\":\"年假\",\"leave_days\":3}"` | 提取字段（JSON 字串） |
| `confidence` | `"0.95"` | 置信度 |
| `reasoning` | `"识别到请假意图，提取到..."` | AI 推理过程 |
| `model` | `"minimax-4"` | 模型名称/版本 |
| `latency_ms` | `"1234"` | 响应耗时（毫秒） |
| `attachments` | `"[\"file1.pdf\",\"file2.jpg\"]"` | 附件列表（JSON 数组） |
| `rating` | `"good"` | 用户评价 |
| `step_type` | `"thinking"` | 处理步骤类型：`thinking` / `reasoning` / `action` |

### 扩展示例

```json
// 意图类型 = form 的消息 metadata
{
  "intent_type": "form",
  "form_code": "leave",
  "extracted_fields": "{\"leave_type\":\"年假\",\"leave_days\":3,\"reason\":\"出差\"}",
  "confidence": "0.97",
  "reasoning": "识别到请假意图，提取到：请假类型=年假，天数=3天"
}

// 意图类型 = chat 的消息 metadata
{
  "intent_type": "chat",
  "model": "minimax-4",
  "latency_ms": "892"
}

// 处理步骤消息（独立消息块）
{
  "step_type": "thinking",
  "intent_type": "form"
}
```

### 处理步骤消息

处理步骤（thinking/reasoning/action）作为**独立消息块**存储在 `chat_messages` 表中，支持完整的会话历史和滚动加载：

| 属性 | 值 | 说明 |
|------|-----|------|
| `role` | `system` | 系统角色 |
| `content_type` | `thinking` | 标识为处理步骤类型 |
| `step_type` (metadata) | `thinking`/`reasoning`/`action` | 具体步骤类型 |
| `parent_id` | 关联消息ID | 可选，关联到触发该步骤的消息 |

**消息顺序示例**：
```
1. user: "帮我填一个请假申请"
2. system: "📋 识别到表单「请假申请」"        (step_type=thinking)
3. system: "📝 从用户输入中提取到 2 个字段"   (step_type=thinking)
4. assistant: "已为您提取以下字段..."         (最终回复)
```

## 五、DDL（MySQL）

```sql
-- ============================================================
-- 聊天会话表
-- ============================================================
CREATE TABLE chat_sessions (
    id                  INTEGER PRIMARY KEY AUTO_INCREMENT COMMENT '自增主键',
    session_id          VARCHAR(64)  NOT NULL UNIQUE             COMMENT '会话唯一标识（UUID）',
    user_id             VARCHAR(100)                            COMMENT '用户ID（可空，支持匿名会话）',
    title               VARCHAR(200)                            COMMENT '会话标题（可由首条消息摘要生成）',
    context_tags        JSON                                    COMMENT '会话标签 ["sales","urgent"] 等',
    session_metadata    JSON                                    COMMENT '会话扩展信息 {source:"web",channel:"api"}',
    status              VARCHAR(20)  DEFAULT 'active'           COMMENT '会话状态 active / archived',
    created_at          DATETIME     DEFAULT CURRENT_TIMESTAMP  COMMENT '创建时间',
    updated_at          DATETIME                                COMMENT '最后消息时间（列表排序用）'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='聊天会话表';

-- ============================================================
-- 聊天消息表（核心字段，与业务完全解耦）
-- ============================================================
CREATE TABLE chat_messages (
    id                  INTEGER PRIMARY KEY AUTO_INCREMENT COMMENT '自增主键',
    message_id          VARCHAR(64)  NOT NULL UNIQUE             COMMENT '消息唯一标识（UUID）',
    session_id          VARCHAR(64)  NOT NULL                    COMMENT '所属会话ID',
    role                VARCHAR(20)  NOT NULL                    COMMENT '角色 user / assistant / system',
    content             TEXT         NOT NULL                    COMMENT '消息正文（支持 Markdown 富文本）',
    content_type        VARCHAR(20)  DEFAULT 'text'              COMMENT '内容类型 text / markdown / json / form',
    parent_id           VARCHAR(64)                             COMMENT '父消息ID（多轮对话树状结构）',
    created_at          DATETIME     DEFAULT CURRENT_TIMESTAMP  COMMENT '创建时间',
    FOREIGN KEY (session_id) REFERENCES chat_sessions(session_id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='聊天消息表（业务字段通过 metadata 表扩展）';

-- ============================================================
-- 消息 KV 扩展表（插入式业务字段，核心设计）
-- ============================================================
CREATE TABLE chat_message_metadata (
    id                  INTEGER PRIMARY KEY AUTO_INCREMENT COMMENT '自增主键',
    message_id          VARCHAR(64)  NOT NULL                    COMMENT '所属消息ID',
    meta_key            VARCHAR(100) NOT NULL                    COMMENT '扩展字段名（intent_type / form_code 等）',
    value               TEXT                                     COMMENT '扩展字段值（JSON 字串或文本）',
    created_at          DATETIME     DEFAULT CURRENT_TIMESTAMP  COMMENT '创建时间',
    FOREIGN KEY (message_id) REFERENCES chat_messages(message_id) ON DELETE CASCADE,
    UNIQUE KEY uq_msg_meta_key (message_id, meta_key)       COMMENT '同消息内 meta_key 不可重复'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='消息 KV 扩展表';

-- ============================================================
-- 索引
-- ============================================================
-- chat_sessions
CREATE INDEX ix_sessions_user_id   ON chat_sessions(user_id);
CREATE INDEX ix_sessions_updated   ON chat_sessions(updated_at DESC);

-- chat_messages
CREATE INDEX ix_messages_session   ON chat_messages(session_id);
CREATE INDEX ix_messages_created   ON chat_messages(created_at ASC);

-- chat_message_metadata
CREATE INDEX ix_meta_message_id    ON chat_message_metadata(message_id);
CREATE INDEX ix_meta_key_value     ON chat_message_metadata(meta_key, value(500));
```

## 六、API 端点

基础路径：`/api/v1/chat`

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/sessions` | 创建会话 |
| GET | `/sessions` | 查询会话列表（按 updated_at 倒序） |
| GET | `/sessions/{session_id}` | 获取会话详情 |
| PATCH | `/sessions/{session_id}` | 更新会话（title / metadata / status） |
| DELETE | `/sessions/{session_id}` | 删除会话（级联删除 messages + metadata） |
| GET | `/sessions/{session_id}/messages` | 获取消息列表（含 metadata） |
| POST | `/sessions/{session_id}/messages` | 保存消息（metadata 在请求体中） |
| GET | `/sessions/{session_id}/messages/{message_id}` | 获取单条消息详情 |
| DELETE | `/sessions/{session_id}/messages/{message_id}` | 删除单条消息 |
| GET | `/messages/search` | 全文搜索消息 |
| GET | `/sessions/{session_id}/stats` | 会话统计（消息数/用户消息数/AI消息数） |

### 请求示例：保存消息

```json
POST /api/v1/chat/sessions/{session_id}/messages
{
  "role": "user",
  "content": "帮我填一个请假申请",
  "content_type": "text",
  "metadata": {
    "intent_type": "form",
    "form_code": "leave",
    "extracted_fields": {"leave_type": "年假", "leave_days": 3},
    "confidence": "0.97"
  }
}
```

### 响应示例：查询消息列表

```json
{
  "messages": [
    {
      "message_id": "uuid-xxx",
      "role": "user",
      "content": "帮我填一个请假申请",
      "content_type": "text",
      "parent_id": null,
      "created_at": "2026-05-07T10:00:00Z",
      "metadata": {
        "intent_type": "form",
        "form_code": "leave",
        "extracted_fields": "{\"leave_type\":\"年假\",\"leave_days\":3}",
        "confidence": "0.97"
      }
    }
  ],
  "total": 1
}
```

## 七、字段命名规范

所有列名均经过 SQL 关键字冲突审查：

| 原拟用名 | 冲突原因 | 最终命名 |
|---------|---------|---------|
| `metadata` | SQLAlchemy 内部 `MetaData` 类名冲突 | `session_metadata`（chat_sessions 表） |
| `session_meta` | 与 SQLAlchemy `metadata` 属性冲突 | `session_metadata` |
| `key` | MySQL 保留关键字 | `meta_key` |
