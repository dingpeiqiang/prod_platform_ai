# 工作流在线传参执行功能说明

## 功能概述

工作流编辑器现已支持在线传参执行功能，允许在运行工作流前动态配置和传递参数。

## 主要特性

### 1. 参数输入面板

新增 `ParameterInputPanel.vue` 组件，提供以下功能：

- **动态添加/删除参数**：可以添加任意数量的执行参数
- **多种数据类型支持**：
  - 字符串（string）
  - 数字（number）
  - 布尔值（boolean）
  - 对象（object，JSON格式）
  - 数组（array，JSON格式）
- **参数描述**：为每个参数添加可选的描述信息
- **实时验证**：自动验证参数名称和值的格式

### 2. 执行引擎增强

更新 `executionEngine.js`，支持：

- **接收输入参数**：`execute(elements, inputParams)` 方法接受第二个参数
- **参数注入上下文**：传入的参数会自动注入到工作流上下文的 `params` 字段
- **变量映射**：参数会自动映射到 `context.variables`，方便后续节点使用
- **执行日志记录**：在执行日志中显示接收到的参数信息

### 3. 两种执行模式

#### 模式一：带参数执行（推荐）

点击工具栏的 **"运行"** 按钮（绿色），会：
1. 打开右侧执行面板
2. 显示参数输入界面
3. 用户可以配置需要的参数
4. 点击"执行工作流"按钮开始执行

#### 模式二：快速执行

点击工具栏的 **"快速执行"** 按钮（灰色），会：
1. 直接执行工作流
2. 不传递任何参数
3. 适用于不需要参数的简单工作流

## 使用方法

### 基本流程

1. **设计工作流**
   - 在画布上拖拽节点并连接
   - 确保有开始节点和结束节点

2. **配置参数**
   - 点击"运行"按钮
   - 在参数面板中添加需要的参数
   - 填写参数名、选择类型、输入值

3. **在工作流中使用参数**
   
   参数可以通过以下方式在节点中使用：

   ```javascript
   // 在提示词节点中使用
   const prompt = "请分析以下数据：{{paramName}}";
   
   // 在条件节点中使用
   const condition = "context.variables.paramName === 'value'";
   
   // 在HTTP节点中使用
   const url = "https://api.example.com?param={{paramName}}";
   ```

4. **执行并查看结果**
   - 点击"执行工作流"
   - 在执行日志中查看参数传递情况
   - 查看执行结果

## 技术实现

### 前端组件

1. **ParameterInputPanel.vue**
   - 位置：`frontend/src/components/workflow-editor/ParameterInputPanel.vue`
   - 功能：参数配置UI
   - 特性：动态表单、类型验证、JSON编辑

2. **LangChainEditor.vue**
   - 集成参数面板
   - 管理执行状态
   - 处理参数传递

3. **executionEngine.js**
   - 位置：`frontend/src/components/workflow-editor/utils/executionEngine.js`
   - 更新：支持接收和使用参数

### 数据流

```
用户输入参数 
    ↓
ParameterInputPanel 验证并转换
    ↓
LangChainEditor 接收参数
    ↓
executionEngine.execute(elements, params)
    ↓
context.params = inputParams
    ↓
context.variables = { ...inputParams }
    ↓
各节点通过 context.variables 访问参数
```

## 注意事项

1. **参数命名规范**
   - 使用英文字母、数字和下划线
   - 避免使用特殊字符
   - 建议使用驼峰命名法

2. **JSON格式验证**
   - 对象和数组类型需要有效的JSON格式
   - 系统会在执行前验证JSON语法
   - 格式错误会阻止执行并提示

3. **参数作用域**
   - 参数在整个工作流执行期间有效
   - 可以在任何节点中通过 `context.variables` 访问
   - 参数值可以被后续节点修改

4. **空参数处理**
   - 如果未配置参数直接执行，`inputParams` 为空对象 `{}`
   - 不会影响工作流的正常执行

## 故障排除

### 问题1：参数无法传递到节点

**原因**：可能使用了错误的变量引用方式

**解决**：
- 确认使用 `context.variables.paramName` 或模板语法 `{{paramName}}`
- 检查参数名是否拼写正确

### 问题2：JSON格式错误

**原因**：对象或数组类型的参数值不是有效的JSON

**解决**：
- 使用JSON验证工具检查格式
- 确保使用双引号而非单引号
- 检查是否有尾随逗号

### 问题3：参数值为空

**原因**：未在参数面板中输入值

**解决**：
- 确保所有必填参数都有值
- 对于可选参数，可以留空但需要在工作流中处理空值情况
