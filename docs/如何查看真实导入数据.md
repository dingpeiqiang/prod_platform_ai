# 如何查看真实的导入数据统计

## 📌 问题说明

如果你在导入后看到"种不同值"而没有具体数字，可能是因为：

1. **前端缓存问题** - 浏览器缓存了旧版本
2. **API返回数据为空** - 后端没有正确返回fieldStats
3. **数据显示问题** - 前端没有正确解析数据

---

## 🔍 调试步骤

### 步骤1: 检查浏览器控制台

1. 打开浏览器开发者工具（F12）
2. 切换到 **Console** 标签
3. 执行导入操作
4. 查看控制台输出，应该看到：

```
=== 导入结果 ===
完整响应: {success: true, message: "...", fieldStats: [...], ...}
fieldStats: [{fieldCode: "applicant_name", distinctValues: 8, topValues: [...]}, ...]
第一个字段统计: {fieldCode: "applicant_name", distinctValues: 8, topValues: [...]}
```

**如果没有看到这些日志**，说明前端代码没有更新，需要刷新页面（Ctrl+F5 强制刷新）。

---

### 步骤2: 检查Network请求

1. 打开浏览器开发者工具（F12）
2. 切换到 **Network** 标签
3. 执行导入操作
4. 找到 `import/execute` 请求
5. 查看 **Response** 标签

应该看到类似这样的JSON响应：

```json
{
  "success": true,
  "message": "成功导入 5 条「leave」历史数据",
  "formCode": "leave",
  "totalImported": 5,
  "totalSource": 5,
  "totalSkipped": 0,
  "totalErrors": 0,
  "fieldStats": [
    {
      "fieldCode": "applicant_name",
      "distinctValues": 4,
      "topValues": [
        {"value": "张三", "count": 2},
        {"value": "李四", "count": 1},
        {"value": "王五", "count": 1},
        {"value": "赵六", "count": 1}
      ]
    },
    {
      "fieldCode": "department",
      "distinctValues": 3,
      "topValues": [
        {"value": "技术部", "count": 3},
        {"value": "销售部", "count": 1},
        {"value": "人事部", "count": 1}
      ]
    }
  ],
  "nestedFields": []
}
```

**如果fieldStats是空数组 `[]`**，说明后端没有返回统计数据。

---

### 步骤3: 测试后端API

运行测试脚本验证API返回：

```bash
# 确保后端服务已启动
cd backend
python -m uvicorn app.main:app --reload --port 8000

# 在另一个终端运行测试
cd ..
python test_api_response.py
```

测试脚本会显示完整的API响应数据结构。

---

### 步骤4: 检查后端日志

查看后端日志文件：

```bash
# Windows
type backend\logs\app.log

# Linux/Mac
tail -f backend/logs/app.log
```

查找包含 `[config/import/execute]` 的日志行，应该看到：

```
[config/import/execute] 开始导入 formCode=leave limit=5
[config/import/execute] 成功导入 5 条「leave」历史数据
```

## ✅ 正确的显示效果

导入成功后，你应该看到：

```
📊 数据统计
┌──────────────────┬──────────┐
│ 源记录数         │ 5        │
│ 实际导入         │ 5        │
│ 跳过(空数据)     │ 0        │
│ 错误数           │ 0        │
└──────────────────┴──────────┘

🔍 字段分布 (Top 5)

▼ applicant_name (4 种不同值)
  ┌──────────────┬────────┐
  │ 值           │ 出现次数│
  ├──────────────┼────────┤
  │ 张三         │ 2 次   │
  │ 李四         │ 1 次   │
  │ 王五         │ 1 次   │
  │ 赵六         │ 1 次   │
  └──────────────┴────────┘

▼ department (3 种不同值)
  ┌──────────────┬────────┐
  │ 技术部       │ 3 次   │
  │ 销售部       │ 1 次   │
  │ 人事部       │ 1 次   │
  └──────────────┴────────┘
```

**注意：**
- ✅ 显示具体的数字（如"4 种不同值"）
- ✅ 显示具体的值和次数（如"张三: 2 次"）
- ❌ 不应该只显示"种不同值"而没有数字

## 🛠️ 常见问题解决

### 问题1: 前端没有显示统计数据

**可能原因：**
- 浏览器缓存了旧版本的前端代码

**解决方案：**
```bash
# 清除浏览器缓存
# 或使用无痕模式打开
# 或强制刷新：Ctrl + F5 (Windows) / Cmd + Shift + R (Mac)
```

---

### 问题2: fieldStats 是空数组

**可能原因：**
- 后端 import_form_data 函数没有正确填充 fieldStats

**解决方案：**
检查后端日志，确认导入过程中是否有错误。

手动测试后端：
```bash
cd backend
python -m app.scripts.import_history leave --limit 5
```

查看输出的"顶层字段分布"部分，确认有数据。

---

### 问题3: 显示"暂无字段统计信息"

**可能原因：**
- API返回的fieldStats为空
- 前端判断条件触发

**解决方案：**
1. 检查Network标签中的API响应
2. 确认fieldStats是否有数据
3. 查看后端日志是否有错误

---

## 📞 仍然有问题？

如果按照以上步骤仍然无法看到正确的统计数据，请提供：

1. **浏览器控制台的完整输出**（Console标签）
2. **API响应的完整JSON**（Network标签 -> Response）
3. **后端日志的相关部分**（backend/logs/app.log）

这样可以帮助快速定位问题。

---

## 💡 提示

- **首次使用建议**：先用命令行测试，确认后端正常工作
- **前端更新后**：务必强制刷新浏览器（Ctrl+F5）
- **查看实时日志**：可以打开两个终端，一个看前端Console，一个看后端日志
