# 智能默认值初始化功能说明

## 功能概述

表单现在支持**从用户配置需求中智能提取信息，并结合推荐值来自动初始化表单默认值**，大大减少用户重复输入！

---

## 核心特性

### 1. 默认值优先级

默认值设置按以下优先级：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 🔴 **第1优先级** | 用户输入提取 | 从用户自然语言中提取的字段值 |
| 🟡 **第2优先级** | 历史推荐（Top1） | 从历史数据中智能推荐的第1个值 |
| 🟢 **第3优先级** | 静态推荐（Top1） | 从配置文件中静态推荐的第1个值 |
| 无 | 为空 | 以上都没有则不设置默认值 |

### 2. 增强的字段提取

支持多种字段提取模式：

#### 提取规则
```
字段名[是为：:] 值
字段名[:：] 值
字段名 值
```

#### 支持的字段（预置）
- 客户姓名、客户电话、手机号
- 订单金额、报销金额、金额
- 订单日期、开始日期、结束日期、费用日期、日期
- 备注、姓名、部门
- 请假类型、请假原因
- 费用类型、费用说明

### 3. 智能类型转换

提取到的值会根据字段类型自动转换：

| 字段类型 | 转换规则 |
|----------|----------|
| **number** | 自动转换为整数或浮点数 |
| **date** | 支持多种格式：`YYYY-MM-DD`、`YYYY/MM/DD`、`YYYY年MM月DD日`，统一输出为 `YYYY-MM-DD` |
| **其他** | 保持原样 |

## 工作流程

```
用户输入自然语言
        ↓
场景识别 → 确定表单类型
        ↓
加载本体Schema
        ↓
加载推荐值（数据库+静态）
        ↓
┌─────────────────────────────────────┐
│  字段提取（LLM优先 → 正则降级）     │
└─────────────────────────────────────┘
        ↓
设置默认值（按优先级）
        ↓
生成带默认值的表单
```

## 使用示例

### 示例1：简单输入，提取多个字段

**用户输入：**
```
帮我填一个销售订单，客户姓名张三，客户电话13800138000，订单金额5000元
```

**处理过程：**
1. 提取到：
   - `customer_name`: "张三"
   - `customer_phone`: "13800138000"
   - `order_amount`: 5000
2. 这些提取到的值设置为默认值
3. 其他字段（如订单日期）使用推荐值第1个

### 示例2：无提取值，使用推荐

**用户输入：**
```
帮我填一个请假申请
```

**处理过程：**
1. 未提取到任何字段值
2. 每个字段的推荐值第1个设为默认值
3. 例如：申请人姓名默认"张三"，部门默认"研发部"

### 示例3：部分提取，部分推荐

**用户输入：**
```
我要请假，姓名李四，请假类型病假
```

**处理过程：**
1. 提取到：
   - `applicant_name`: "李四"
   - `leave_type`: "病假"
2. 其他字段使用推荐值

## 实现细节

### 核心文件修改

| 文件 | 说明 |
|------|------|
| `app/skills/field_extraction.py` | 增强字段提取，支持更多模式、智能类型转换 |
| `app/services/form_service.py` | 优化默认值逻辑，按优先级设置默认值 |

### API响应增强

表单生成接口现在返回：
```json
{
  "success": true,
  "formSchema": { ... },
  "formId": "form_xxx",
  "sceneMethod": "regex",
  "extractedFields": ["customer_name", "customer_phone"]
}
```

新增字段：`extractedFields` - 列出从用户输入中成功提取的字段

## 扩展建议

1. **支持同义词提取**：例如"客户"="顾客"，"金额"="钱"
2. **支持模糊匹配**：例如"张三"也能匹配"张先生"
3. **支持上下文理解**：记住上次用户输入的值
4. **支持单位自动转换**：例如"5千"→"5000"
5. **支持多语言**：中英文混合输入也能识别
