# 🚀 后端启动脚本 - 快速参考

## 📌 基本用法

### 生产模式（默认）
```bash
start-backend.bat
```
或双击运行

**特点**:
- ⚡ 快速启动
- 📝 完整日志
- 🔒 稳定可靠

---

### 开发模式
```bash
start-backend.bat dev
```

**特点**:
- 🔄 代码修改自动重启
- 💻 适合开发调试
- 📄 日志保存在文件

---

## 📊 启动信息

### 访问地址
- **API服务**: http://localhost:6173
- **API文档**: http://localhost:6173/docs
- **健康检查**: http://localhost:6173/health

### 端口说明
- 默认端口: **6173**
- 如果被占用: 自动切换到 **6174**

### 日志位置
- **终端**: 实时显示
- **文件**: `backend/logs/app.log`

## 💡 使用场景

### 场景1: 日常开发
```bash
# 直接启动，无需任何操作
start-backend.bat
```

### 场景2: 修改代码调试
```bash
# 启用热重载，修改代码后自动重启
start-backend.bat dev
```

```batch
@echo off
start start-backend.bat
timeout /t 5
curl http://localhost:6173/health
```

## ❓ 常见问题

### Q1: 如何停止服务？
按 `Ctrl+C` 或关闭窗口

### Q2: 端口被占用怎么办？
脚本会自动切换到下一个可用端口（6173 → 6174）

### Q3: 首次启动很慢？
首次运行会创建虚拟环境并安装依赖，需要几分钟时间

### Q4: 如何查看完整日志？
打开文件：`backend/logs/app.log`
