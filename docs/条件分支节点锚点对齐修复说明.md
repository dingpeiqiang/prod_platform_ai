# 条件分支节点锚点对齐修复说明

## 问题描述

条件分支节点(ConditionNode)上的彩色锚点(Handle)与对应的条件分支不对齐,导致连线时视觉上不匹配。

## 根本原因

1. **缺少 compact 属性**: ConditionNode 在画布模板中没有传递 `compact` 属性,导致始终显示完整内容而非紧凑模式
2. **位置计算逻辑不完善**: `calculateHandlePositions()` 函数只支持非配置模式下的 `.branch-condition-item` 元素,在配置模式下找不到目标元素
3. **DOM 元素缺失**: 配置模式下没有对应的箭头元素用于定位

## 修复方案

### 1. 添加 compact 属性 (LangChainEditor.vue)

**文件**: `frontend/src/components/workflow-editor/LangChainEditor.vue`  
**位置**: 第 362-368 行

```vue
<template #node-condition="props">
  <ConditionNode
    :data="enrichNodeData(props.data, props.node?.id)"
    :selected="props.selected"
    compact  <!-- ✅ 新增 -->
    @update="updateNodeData"
  />
</template>
```

**效果**: 确保条件分支节点在画布上以紧凑模式显示,与其他节点保持一致的UI风格。

### 3. 添加配置模式监听 (ConditionNode.vue)

**文件**: `frontend/src/components/workflow-editor/nodes/ConditionNode.vue`  
**位置**: 第 520-530 行

```javascript
// 监听配置模式变化，确保在两种模式下都能正确定位锚点
const configModeRef = ref(props.configMode);
watch(() => props.configMode, (newMode) => {
  configModeRef.value = newMode;
  // 配置模式切换时延迟计算，等待DOM更新
  setTimeout(() => {
    calculateHandlePositions();
  }, 50);
});
```

**效果**: 当节点在配置模式和画布模式之间切换时,自动重新计算锚点位置。

```javascript
const updateNodeData = (nodeId, data) => {

  const node = elements.value.find(el => el.id === nodeId);

  }
};
```

**效果**: 当条件分支节点的配置数据更新时,触发重新渲染以确保锚点位置正确更新。

## 技术要点

### Vue Flow Handle 定位机制

Vue Flow 的 Handle 组件支持通过 `style` 属性自定义位置:

```vue
<Handle
  type="source"
  :position="Position.Right"
  :id="'branch_' + index"
  :style="{ top: getHandlePosition(index) + '%' }"
/>
```

- `position`: 确定 Handle 在节点的哪一侧(Left/Right/Top/Bottom)
- `style.top`: 覆盖默认的垂直位置,使用百分比定位

### ResizeObserver 监听

使用 `ResizeObserver` 监听节点尺寸变化,确保在节点大小改变时重新计算锚点位置:

```javascript
const resizeObserver = new ResizeObserver(() => {
  calculateHandlePositions();
});
resizeObserver.observe(nodeRef.value);
```

### 防抖处理

位置计算使用 `nextTick()` 和 `setTimeout()` 确保 DOM 已更新后再执行计算,避免获取到错误的尺寸信息。
