## 1. 项目概述

### 1.1 项目简介
AI驱动动态表单底层框架是一个聚焦于聊天窗口场景的表单系统，通过自然语言交互生成表单，支持实时双向通信、本体约束校验、历史数据推荐等功能。

### 1.2 核心特性
- **全配置化架构**：所有业务逻辑、参数、本体均可通过配置文件管理
- **通用聊天助手**：既能聊天又能填表单
- **AI表单生成**：自然语言输入 → 场景识别 → Schema生成
- **流式推理过程**：SSE流式输出，实时显示思考过程
- **聊天窗口适配**：动态组件渲染、推荐值提示
- **双向可控**：WebSocket实时通信、版本号冲突解决
- **本体约束**：业务规则定义、Schema合规校验
- **历史推荐**：智能推荐、减少重复输入
- **LLM集成**：支持OpenAI/本地模型，智能降级到规则
- **基于Skills和本体**：LLM基于本体和Skills做需求解析和意图识别
- **自动化校验**：12种内置规则 + 自定义校验
- **外部API调用**：枚举、校验、提交API配置化

### 1.3 技术栈
| 层级 | 技术 |
|------|------|
| 后端 | FastAPI, SQLAlchemy, Pydantic |
| 前端 | Vue 3, Element Plus, Vite |
| 数据库 | SQLite（可扩展为MySQL） |
| 实时通信 | WebSocket |
| 流式输出 | Server-Sent Events (SSE) |
| LLM | OpenAI API兼容 |

## 3. 环境配置

### 3.1 后端环境配置

#### 3.1.1 安装依赖
```bash
cd backend
pip install -r requirements.txt
```

#### 3.1.2 环境变量配置
复制 `.env.example` 为 `.env` 并配置：

```env
# 应用配置
APP_NAME=AI驱动动态表单底层框架
APP_VERSION=2.1.0
DEBUG=true

# WebSocket配置
WEBSOCKET_HOST=0.0.0.0
WEBSOCKET_PORT=8000

# 数据库配置
DATABASE_URL=sqlite:///./test.db

# LLM配置（可选）
LLM_API_KEY=your_api_key_here
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4
```

### 3.2 前端环境配置
```bash
cd frontend
npm install
```

## 4. 配置系统详解

### 4.1 配置文件总览

| 配置文件 | 说明 |
|---------|------|
| `config/app_config.json` | 应用主配置（LLM等） |
| `config/system_config.json` | 系统参数（推荐权重、分隔符等） |
| `config/ontologies/*.json` | 业务本体定义 |
| `config/scenes/scene_mapping.json` | 场景映射配置 |
| `config/templates/recommendations.json` | 静态推荐值 |
| `config/prompts/*.txt` | LLM提示词模板 |

### 4.2 app_config.json详解
```json
{
  "llm": {
    "enabled": true,
    "apiKey": "your-api-key",
    "baseUrl": "https://api.openai.com/v1",
    "model": "gpt-4",
    "temperature": 0.3,
    "maxTokens": 2048,
    "fallbackToRules": true
  }
}
```

### 4.3 system_config.json详解

#### 推荐参数
```json
{
  "recommendation": {
    "countScoreWeight": 0.4,
    "userScoreWeight": 0.4,
    "timeScoreWeight": 0.2,
    "countScorePerUnit": 0.1,
    "userScorePerUnit": 0.2,
    "timeDecayDays": 30,
    "historyQueryLimit": 1000,
    "recommendationLimit": 3
  },
  "sceneRecognition": {
    "defaultScene": "generic"
  },
  "fieldExtraction": {
    "separators": ["是", "为", "：", ":", " "]
  }
}
```

### 4.4 配置热加载API
```bash
POST /api/v1/config/reload
```
可选参数：`config_type`（all, system_config, app_config, ontologies等）

## 5. 开发指南

#### 添加新API
1. 在 `app/api/` 下创建新路由文件
2. 在 `app/schemas/form.py` 中定义请求/响应模型
3. 在 `app/main.py` 中注册路由

修改配置文件后调用热加载API，无需重启服务

## 6. 常见问题排查

### 6.1 字段名冲突问题
**问题**：Pydantic字段名"schema"或SQLAlchemy字段名"metadata"冲突
**解决**：修改字段名为form_schema或log_metadata

### 6.2 数据库连接问题
**问题**：数据库连接失败
**解决**：检查 `.env` 中的 `DATABASE_URL`

### 6.3 LLM调用失败
**问题**：LLM调用失败，自动降级到规则
**解决**：检查 `app_config.json` 中的 `fallbackToRules` 为 true

### 6.4 配置修改不生效
**问题**：修改配置后未生效
**解决**：调用热加载API `/api/v1/config/reload`

## 7. 扩展开发指南

### 7.1 添加新业务场景
无需修改代码，只需：
1. 在 `config/ontologies/` 创建新的本体JSON
2. 在 `config/scenes/scene_mapping.json` 添加场景映射
3. 调用热加载API

### 7.2 添加新校验规则
内置12种规则，自定义校验：
- **方式1：simple表达式
- **方式2：lambda函数
- **方式3：代码注册

### 7.3 配置外部API
枚举API、校验API、提交API都可配置

### 7.4 调整推荐权重
修改 `config/system_config.json` 中的推荐参数
