# 架构改进总结 - v2.0

## 改进概述

针对v1.0版本中硬编码严重、AI能力缺失的问题，v2.0版本进行了全面重构，实现了**全配置化架构**和**真实LLM AI能力**集成。

---

## 核心改进点

### 1. ✅ 全配置化架构

#### 1.1 配置目录结构
```
backend/config/
├── app_config.json          # 应用主配置
├── ontologies/              # 本体定义（JSON）
│   ├── sales_order.json
│   ├── leave.json
│   └── expense.json
├── scenes/                  # 场景配置
│   └── scene_mapping.json
├── templates/               # 模板配置
│   └── recommendations.json
└── prompts/                 # LLM提示词
    ├── intent_recognition.txt
    └── field_extraction.txt
```

#### 1.2 配置加载器 (`app/core/config_loader.py`)
- **单例模式**：全局唯一配置加载器
- **缓存机制**：配置加载后缓存，避免重复IO
- **热加载支持**：运行时可重新加载配置
- **多格式支持**：JSON、TXT等格式

**核心API：**
```python
config_loader.get_app_config()           # 获取应用配置
config_loader.get_ontology(form_code)    # 获取指定本体
config_loader.get_all_ontologies()       # 获取所有本体
config_loader.get_recommendations(...)    # 获取推荐值
config_loader.get_prompt(prompt_name)    # 获取提示词
config_loader.reload_config(type)         # 热加载配置
```

#### 1.3 本体配置化
- 本体从代码迁移到 `config/ontologies/` 目录
- JSON格式，易于编辑和版本管理
- 支持动态热加载，无需重启服务

**本体示例：**
```json
{
  "formCode": "sales_order",
  "formName": "销售订单",
  "entities": [
    {
      "entityCode": "customer",
      "fields": [...]
    }
  ]
}
```

#### 1.4 场景映射配置化
- 场景关键词映射从代码迁移到 `config/scenes/scene_mapping.json`
- 支持优先级配置
- 支持灵活的关键词扩展

---

### 2. ✅ 真实LLM AI能力

#### 2.1 LLM服务 (`app/services/llm_service.py`)
- **多提供商支持**：OpenAI、兼容OpenAI格式的本地模型
- **配置化参数**：模型、温度、Token数均可配置
- **降级机制**：LLM不可用时自动降级到规则匹配
- **结构化输出保证**：JSON提取机制

**核心能力：**
```python
llm_service.recognize_intent(user_input, form_types)  # 意图识别
llm_service.extract_fields(user_input, form_schema)   # 字段提取
```

#### 2.2 提示词工程
- 提示词独立存储在 `config/prompts/` 目录
- 支持模板变量替换
- 易于优化和A/B测试

**意图识别提示词示例：**
```
你是一个专业的表单意图识别助手。请从用户输入中识别出用户想要填写的表单类型。

可用的表单类型：
{form_types}

请严格按照以下JSON格式输出...
```

#### 2.3 Skills插件增强
- **场景识别**：优先LLM → 降级关键词 → 默认场景
- **字段提取**：优先LLM → 降级正则 → 返回空
- **方法标识**：返回结果中包含 `method` 字段，便于调试

---

### 3. ✅ 动态配置管理API

新增配置管理API路由 (`app/api/config.py`)：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/config/reload` | POST | 热加载配置 |
| `/api/v1/config/app` | GET | 获取应用配置 |
| `/api/v1/config/ontologies` | GET | 获取本体列表 |

**配置热加载使用：**
```bash
# 重新加载所有配置
curl -X POST http://localhost:8000/api/v1/config/reload \
  -H "Content-Type: application/json" \
  -d '{"configType": "all"}'

# 仅重新加载本体
curl -X POST http://localhost:8000/api/v1/config/reload \
  -H "Content-Type: application/json" \
  -d '{"configType": "ontologies"}'
```

### 4. ✅ 架构优化

#### 4.1 服务层重构
- `OntologyService`：从配置加载器读取本体
- `HistoryService`：从配置加载器读取推荐值
- `FormService`：支持LLM方法标识返回

#### 4.2 应用配置 (`config/app_config.json`)
```json
{
  "app": { "name": "...", "version": "2.0.0" },
  "llm": {
    "enabled": true,
    "provider": "openai",
    "model": "gpt-4",
    "temperature": 0.3,
    "fallbackToRules": true
  },
  "features": {
    "enableHotReload": true,
    "enableCache": true
  }
}
```

## 使用指南

### 启用LLM能力

1. 复制 `.env.example` 为 `.env`
2. 配置LLM参数：
```env
LLM_API_KEY=your_api_key_here
LLM_BASE_URL=  # 可选，用于本地模型
LLM_MODEL=gpt-4
```

3. 在 `config/app_config.json` 中启用：
```json
{
  "llm": {
    "enabled": true,
    "fallbackToRules": true
  }
}
```

### 添加新的业务场景

1. 在 `config/ontologies/` 创建新的JSON文件，如 `contract.json`
2. 在 `config/scenes/scene_mapping.json` 添加场景映射
3. （可选）在 `config/templates/recommendations.json` 添加推荐值
4. 调用热加载API或重启服务

### 自定义提示词

编辑 `config/prompts/` 目录下的提示词文件，然后调用热加载API。

---

## 迁移指南

### 从v1.0迁移到v2.0

1. **本体迁移**：将硬编码的本体数据迁移到 `config/ontologies/` 目录
2. **场景映射迁移**：将场景关键词迁移到 `config/scenes/scene_mapping.json`
3. **推荐值迁移**：将推荐数据迁移到 `config/templates/recommendations.json`
4. **配置LLM**：按需配置LLM参数（可选）
5. **测试验证**：确保所有功能正常工作

---

## 总结

v2.0版本的核心改进：

| 方面 | v1.0 | v2.0 |
|------|------|------|
| **本体管理** | 硬编码在代码中 | JSON配置文件 |
| **场景映射** | 硬编码在代码中 | JSON配置文件 |
| **推荐数据** | 硬编码在代码中 | JSON配置文件 |
| **AI能力** | 仅规则匹配 | LLM+规则双重支持 |
| **提示词** | 无 | 可配置提示词 |
| **配置管理** | 需重启服务 | 支持热加载 |
| **扩展性** | 需修改代码 | 仅需修改配置 |

v2.0版本已具备**企业级通用框架能力**，可灵活扩展，支持智能化应用！
