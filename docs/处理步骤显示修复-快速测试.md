# 处理步骤显示修复 - 快速测试指南

## 🎯 修复内容

**问题**：看到 loading 消息（"正在为你生成表单..."），但看不到模型思考日志（处理步骤）。

**原因**：loading 消息是前端独立创建的，没有 reasoning 数组。

**解决**：让 loading 消息从上一个流式消息中继承 reasoning 数组。

## ✅ 修改的文件

1. **backend/app/intent/handlers/form_handler.py**
   - 添加最终提示步骤："✅ 准备生成 xxx 表单..."

2. **frontend/src/composables/useFormHandling.js** ⭐
   - loading 消息继承上一个消息的 reasoning 数组
   - 自动展开 reasoning 面板（如果有内容）
   - 数据库保存时包含 reasoning

## 🔧 常见问题

### Q1: 看到了处理步骤，但是没有自动展开？

**检查**：`showReasoning` 是否为 `true`

**解决**：确认代码中有这行：
```javascript
showReasoning: inheritedReasoning.length > 0
```

### Q2: 处理步骤是空的？

**检查**：`reasoning` 数组是否为空

**可能原因**：
- 后端没有发送 thinking 事件
- 前端没有正确处理 thinking 事件

**解决**：检查后端日志和 Network 标签

### Q3: 看不到 Prompt/Response？

**检查**：是否收到 `reasoning` 类型的事件

**可能原因**：
- 后端没有调用 `yield reasoning(...)`
- 导入语句不正确

**解决**：检查 form_handler.py 中是否有：
```python
from ..utils import reasoning as reasoning_event
yield reasoning_event(...)
```
