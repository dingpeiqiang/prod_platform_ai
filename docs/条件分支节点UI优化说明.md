# 条件分支节点UI优化说明

## 优化概述

参照开始节点、结束节点、LLM节点的UI风格,对条件分支节点进行了全面优化,实现了统一的视觉风格和交互体验。

## 主要改动

### 1. 头部组件标识区

**设计特点:**
- **品牌图标**:橙色渐变方形背景(`linear-gradient(135deg, #ff7a45 0%, #fa541c 100%)`),内有白色抽象箭头/闪电样式的SVG图标
- **组件名称输入框**:白色输入框,支持自定义组件名称,默认文字"判断器"
- **功能说明文字**:浅灰色(#999)说明文本,清晰解释判断器的核心逻辑

**代码实现:**
```vue
<div class="component-header">
  <div class="brand-icon">
    <svg width="20" height="20" viewBox="0 0 24 24" fill="none" stroke="white" stroke-width="2">
      <path d="M7 17l9.2-9.2M17 17V7H7"/>
    </svg>
  </div>
  <input 
    v-model="localLabel" 
    @input="emitUpdate"
    type="text"
    placeholder="判断器"
    class="component-name-input"
  />
  <p class="component-description">
    连接两个下游分支,如果设立的条件成立,则只运行如果分支,不成立则只运行否则分支
  </p>
</div>
```

### 2. 主体「条件分支」配置区

#### 标题与全局操作
- **左侧标题**:加粗黑色文字"条件分支"
- **右侧操作按钮**:蓝色文字+图标"新增分支",点击可添加更多条件分支

#### 「如果」分支配置项(第一优先级条件)
- **分支标题**:黑色文字"如果",右侧带有帮助图标(i)
- **条件配置行**(四列布局):
  - **条件**:文字标签,无交互控件
  - **引用变量**:下拉选择框,占位文字"请选择...",用于选择流程中已定义的变量
  - **选择条件**:下拉选择框,占位文字"请选择条件",包含等于、大于、包含等运算符
  - **比较值**:组合控件
    - 左侧下拉选择框(默认选中"输入",支持切换为"引用变量")
    - 右侧文本输入框,占位文字"请输入"
- **右侧操作按钮**:蓝色+号图标,为当前分支添加更多条件

#### 「否则如果」分支配置项(第二优先级条件)
- **分支标题**:黑色文字"否则如果",同样带有帮助图标
- **条件配置行**:与「如果」分支完全一致
- **右侧操作按钮**:
  - 蓝色×号:删除当前「否则如果」分支
  - 蓝色+号:为当前分支添加更多条件

### 3. 底部操作区

- **居中的橙色矩形按钮**:文字为"收起",点击可折叠当前的条件分支配置面板

## 数据结构设计

### 分支数据结构
```javascript
{
  branches: [
    {
      type: 'if',           // 分支类型: if / else_if
      expanded: true,       // 是否展开
      conditions: [         // 条件列表
        {
          variable: '',     // 引用变量ID
          operator: '',     // 运算符: ==, !=, >, <, contains等
          valueType: 'input', // 值类型: input / reference
          value: ''         // 比较值或引用变量ID
        }
      ]
    },
    {
      type: 'else_if',
      expanded: true,
      conditions: [...]
    }
  ]
}
```

### 初始化逻辑
- 首次创建时自动初始化两个分支:"如果"和"否则如果"
- 每个分支默认包含一个空条件
- 触发更新保存默认结构

## 交互特性

### 可扩展性
1. **多分支支持**:通过"+新增分支"按钮添加多个「否则如果」分支,实现多条件、多分支的流程控制
2. **多条件组合**:每个分支内支持通过+号添加多个条件,实现"且/或"的逻辑组合判断

### 用户友好性
1. **清晰的占位文字**:所有配置项都有明确的占位提示
2. **帮助图标**:每个分支提供额外说明,降低使用门槛
3. **折叠功能**:"收起"按钮可折叠复杂配置,减少界面干扰

### 视觉层级
1. **标题突出**:标题文字加粗,功能说明文字浅灰色弱化
2. **操作区分**:添加按钮(橙色)与删除按钮(×)采用不同的视觉符号,避免误操作
3. **状态反馈**:悬停效果、焦点状态提供清晰的交互反馈

## 样式规范

### 颜色方案
- **主色调**:橙色系 `#ff7a45` (hover: `#fa541c`)
- **辅助色**:蓝色系用于操作按钮
- **背景色**:白色 `#ffffff`,浅灰 `#fafafa`
- **边框色**: `#e8e8e8`, `#d9d9d9`
- **文字色**:深灰 `#333`,中灰 `#666`,浅灰 `#999`

### 圆角规范
- 大容器:8px
- 中等元素:6px
- 小元素:4px
- 圆形按钮:50%

### 间距规范
- 区块内边距:16px
- 元素间距:8px
- 紧凑间距:4px

### 动画效果
- 展开/折叠:`slideDown`动画,0.3s缓动
- 悬停过渡:0.2s ease
- 按钮点击:scale(0.98)

## 技术实现要点

### Vue Composition API
```javascript
const localBranches = ref([]);
const branchSectionExpanded = ref(true);

// 初始化分支数据
const initBranches = () => {
  if (props.data.branches && props.data.branches.length > 0) {
    localBranches.value = JSON.parse(JSON.stringify(props.data.branches));
  } else {
    // 默认创建两个分支
    localBranches.value = [...];
    emitUpdate();
  }
};
```

### 响应式更新
- 所有用户操作都触发`emitUpdate()`
- 使用`watch`监听props.data变化,同步本地状态
- 深度监听确保嵌套对象变化也能被捕获

### 可用变量集成
- 接收`availableVariables` prop
- 在变量选择下拉框中展示可用的流程变量
- 支持引用其他节点的输出作为判断条件
