# 空会话唯一性测试指南

## 测试目标

验证系统中只会存在一个空会话（没有 dbSessionId 的会话）。

## 测试步骤

### 测试1：多次点击"新建会话"按钮

1. **初始状态**：登录系统，确保当前没有任何会话或只有一个空会话

2. **操作步骤**：
   - 点击侧边栏的"新建会话"按钮
   - 再次点击"新建会话"按钮
   - 第三次点击"新建会话"按钮

3. **预期结果**：
   - ✅ 侧边栏中只显示一个"新对话"会话
   - ✅ 每次点击都切换到同一个空会话
   - ✅ 控制台显示：`[App] 发现空会话，切换到: xxx`

4. **验证方法**：
   - 打开浏览器开发者工具（F12）
   - 查看 Console 标签页
   - 观察日志输出
   - 检查 localStorage 中的 `chat_sessions` 数据

### 测试2：初始化时清理多余空会话

1. **准备条件**：手动在 localStorage 中创建多个空会话

2. **操作步骤**：
   ```javascript
   // 在浏览器控制台中执行
   const sessions = [
     { id: 'sess_1', title: '新对话', createdAt: Date.now() - 3000, updatedAt: Date.now() - 3000, dbSessionId: null },
     { id: 'sess_2', title: '新对话', createdAt: Date.now() - 2000, updatedAt: Date.now() - 2000, dbSessionId: null },
     { id: 'sess_3', title: '新对话', createdAt: Date.now() - 1000, updatedAt: Date.now() - 1000, dbSessionId: null }
   ]
   localStorage.setItem('chat_sessions', JSON.stringify(sessions))
   location.reload() // 刷新页面
   ```

3. **预期结果**：
   - ✅ 页面加载后，只保留最新的一个空会话（sess_3）
   - ✅ 控制台显示：`[App] 发现 3 个空会话，只保留最新的一个`
   - ✅ 控制台显示：`[App] 删除多余空会话: sess_1` 和 `[App] 删除多余空会话: sess_2`
   - ✅ localStorage 中只剩下一个空会话

4. **验证方法**：
   - 查看 Console 日志
   - 检查 localStorage 中的 `chat_sessions`
   - 确认只有最新的空会话被保留

### 测试3：空会话发送消息后创建新会话

1. **初始状态**：有一个空会话

2. **操作步骤**：
   - 在空会话中发送一条消息：“帮我填写销售订单”
   - 等待AI回复完成
   - 点击“新建会话”按钮

3. **预期结果**：
   - ✅ 第一个会话现在有消息，应该有 dbSessionId
   - ✅ 第一个会话的标题应该是“帮我填写销售订单”（前20个字符）
   - ✅ 点击“新建会话”后创建了一个新的空会话
   - ✅ 现在有两个会话：一个有消息的会话 + 一个空会话
   - ✅ 控制台显示：`[App] 创建新的空会话`

4. **验证方法**：
   - 检查侧边栏显示两个会话
   - 查看 localStorage 中的 `chat_sessions`
   - 确认第一个会话有 dbSessionId 和有意义的标题
   - 确认第二个会话没有 dbSessionId，标题为“新对话”

### 测试4：重新登录后空会话唯一性

1. **初始状态**：有多个会话（包括一个空会话）

2. **操作步骤**：
   - 退出登录
   - 重新登录

3. **预期结果**：
   - ✅ 所有有消息的会话都正确加载
   - ✅ 如果存在多个空会话，只保留最新的一个
   - ✅ 控制台显示清理多余空会话的日志

4. **验证方法**：
   - 查看 Console 日志
   - 检查侧边栏会话列表
   - 确认只有一个空会话

## 常见问题排查

### 问题1：仍然出现多个空会话

**可能原因**：
- 浏览器缓存未清除
- localStorage 数据损坏

**解决方法**：
```javascript
// 在控制台中执行
localStorage.clear()
location.reload()
```

### 问题2：空会话没有被正确识别

**可能原因**：
- dbSessionId 字段不是 null 而是空字符串或其他值

**检查方法**：
```javascript
// 在控制台中执行
const sessions = JSON.parse(localStorage.getItem('chat_sessions'))
console.log(sessions.map(s => ({ id: s.id, dbSessionId: s.dbSessionId, type: typeof s.dbSessionId })))
```

**预期输出**：
```javascript
[
  { id: 'sess_xxx', dbSessionId: null, type: 'object' },  // 空会话
  { id: 'sess_yyy', dbSessionId: 'uuid...', type: 'string' }  // 有消息的会话
]
```

### 问题3：控制台没有显示相关日志

**可能原因**：
- 代码未正确部署
- 浏览器缓存了旧版本的 JavaScript 文件

**解决方法**：
- 硬刷新页面（Ctrl + Shift + R 或 Cmd + Shift + R）
- 清除浏览器缓存
- 确认前端服务已重启

## 自动化测试脚本

可以在浏览器控制台中运行以下脚本进行快速测试：

```javascript
// 测试空会话唯一性
function testEmptySessionUniqueness() {
  console.log('=== 开始测试空会话唯一性 ===')
  
  // 1. 获取当前会话列表
  const sessions = JSON.parse(localStorage.getItem('chat_sessions') || '[]')
  console.log('当前会话数量:', sessions.length)
  
  // 2. 统计空会话数量
  const emptySessions = sessions.filter(s => !s.dbSessionId)
  console.log('空会话数量:', emptySessions.length)
  
  // 3. 验证结果
  if (emptySessions.length <= 1) {
    console.log('✅ 测试通过：空会话数量符合要求（≤ 1）')
  } else {
    console.log('❌ 测试失败：存在多个空会话')
    console.log('空会话列表:', emptySessions)
  }
  
  return emptySessions.length <= 1
}

// 运行测试
testEmptySessionUniqueness()
```

## 总结

通过以上测试，可以验证：
1. ✅ 系统中只会存在一个空会话
2. ✅ 多次点击“新建会话”不会创建多个空会话
3. ✅ 初始化时会自动清理多余的空会话
4. ✅ 空会话发送消息后可以创建新的空会话
5. ✅ 重新登录后空会话唯一性得到保持
6. ✅ **首次创建数据库会话时，使用用户的第一条消息内容作为标题**
