# 基于本体的字段提取功能说明

## 改进概述

字段提取现在已重构为**完全基于本体驱动**，不再使用硬编码的字段映射！

---

## 核心改进

| 方面 | 之前（v1） | 现在（v2） |
|------|-----------|-----------|
| **字段映射 | 硬编码 `FIELD_PATTERNS` | 从本体动态读取 |
| **扩展性** | 需修改代码 | 仅需修改配置 |
| **维护性** | 需同步代码维护 | 与本体配置维护 |

---

## 工作原理

### 流程
```
用户输入自然语言
        ↓
场景识别 → 确定表单类型
        ↓
加载本体 Schema
        ↓
遍历本体中的所有字段
        ↓
对每个字段名动态生成正则
        ↓
尝试匹配用户输入
        ↓
智能类型转换
        ↓
返回提取到的字段
```

### 支持的分隔符
```
字段名[是为：:] 值
字段名[:：] 值
字段名 值
```

支持的分隔符：`是`、`为`、`：`、`:`、` `（空格）

## 使用示例

### 配置销售订单本体
```json
{
  "formCode": "sales_order",
  "entities": [
    {
      "entityCode": "customer",
      "fields": [
        { "fieldCode": "customer_name", "fieldName": "客户姓名", "fieldType": "input" },
        { "fieldCode": "customer_phone", "fieldName": "客户电话", "fieldType": "input" },
        { "fieldCode": "order_amount", "fieldName": "订单金额", "fieldType": "number" }
      ]
    }
  ]
}
```

### 用户输入
```
帮我填一个销售订单，客户姓名张三，客户电话13800138000，订单金额5000元
```

### 提取过程
1. 从本体读取字段：
   - 客户姓名 → customer_name
   - 客户电话 → customer_phone
   - 订单金额 → order_amount

2. 对每个字段名动态生成正则：
   - `客户姓名[是为：:]\s*([^\s，,。]+)`
   - `客户电话[是为：:]\s*([^\s，,。]+)`
   - `订单金额[是为：:]\s*([^\s，,。]+)`

3. 匹配用户输入，成功提取：
   - customer_name: "张三"
   - customer_phone: "13800138000"
   - order_amount: 5000

## 扩展指南

### 如何添加新的可提取字段
**只需三步！**
1. 在 `config/ontologies/` 中添加或修改本体JSON
2. 添加字段定义（包含 fieldName、fieldCode、fieldType）
3. 重启服务或调用热加载API

**无需修改任何代码！

### 本体示例
```json
{
  "fieldCode": "new_field",
  "fieldName": "新字段名称",
  "fieldType": "input"
}
```

然后用户就可以输入：
```
新字段名称是我的值
```

系统会自动识别并提取！

## 类型转换增强

| 字段类型 | 转换规则 |
|----------|----------|
| **number** | 自动清理 `元￥$，, 等符号，转换为整数/浮点数 |
| **date** | 支持 `YYYY-MM-DD`、`YYYY/MM/DD`、`YYYY年MM月DD日`，统一为 `YYYY-MM-DD` |
| **其他** | 保持原样 |
