# ✅ 编码存储方案 - 实施完成

## 🎯 核心原则

**数据库存储编码，用户界面显示中文**

```
┌─────────────────────────┐
│  UI 显示层              │
│  "年假"、"病假"、"事假"  │  ← 用户看到中文
└───────────┬─────────────┘
            ↓ 映射
┌─────────────────────────┐
│  应用逻辑层             │
│  annual, sick, personal │  ← 代码使用编码
└───────────┬─────────────┘
            ↓ 存储
┌─────────────────────────┐
│  数据库层               │
│  annual, sick, personal │  ← 数据库存编码
└─────────────────────────┘
```

**文件：** `backend/app/services/history_service.py`

**修改内容：**

#### a) 添加OntologyService导入

```python
from app.services.ontology_service import OntologyService
```

#### b) 修改get_recommend_values返回格式
```python
# ❌ 之前：只返回值
top_recommendations = [r['value'] for r in sorted_recommendations[:limit]]
# 返回：["年假", "病假"]

# ✅ 现在：返回编码和标签
for r in sorted_recommendations[:limit]:
    code = r['value']  # 编码
    label = cls._get_field_label(form_code, field_code, code)  # 中文
    top_recommendations.append({
        'value': code,    # 用于提交
        'label': label,   # 用于显示
        'score': r['score'],
        'source': r.get('source', 'db')
    })
# 返回：[{value: "annual", label: "年假"}, ...]
```

#### c) 新增_get_field_label方法
```python
@classmethod
def _get_field_label(cls, form_code: str, field_code: str, code: str) -> str:
    """从本体定义中获取字段的中文标签"""
    try:
        ontology_result = OntologyService.get_form_constraint(form_code)
        if not ontology_result.get('success'):
            return code
        
        constraints = ontology_result.get('constraints', {})
        fields = constraints.get('fields', {})
        field_def = fields.get(field_code, {})
        
        options = field_def.get('options', [])
        for opt in options:
            if opt.get('value') == code:
                return opt.get('label', code)
        
        return code  # 没有映射则返回原值
    except Exception as e:
        logger.warning("[HistoryService] 获取字段标签失败: %s", e)
        return code
```

#### b) 新增辅助函数

```javascript
// 获取选项的值（支持 {value, label} 格式和纯字符串）
const getOptionValue = (option) => {
  if (typeof option === 'object' && option !== null) {
    return option.value !== undefined ? option.value : option
  }
  return option
}

// 获取选项的标签（支持 {value, label} 格式和纯字符串）
const getOptionLabel = (option) => {
  if (typeof option === 'object' && option !== null) {
    return option.label !== undefined ? option.label : (option.value || String(option))
  }
  return String(option)
}
```

**兼容性：**
- ✅ 支持新格式：`{value: "annual", label: "年假"}`
- ✅ 兼容旧格式：`"年假"`（纯字符串）

## 📊 数据流示例

### 场景：用户填写请假表单

#### 步骤1: AI生成表单

后端返回Schema：
```json
{
  "fieldCode": "leave_type",
  "fieldName": "请假类型",
  "fieldType": "select",
  "options": [
    {"value": "annual", "label": "年假"},
    {"value": "sick", "label": "病假"},
    {"value": "personal", "label": "事假"}
  ]
}
```

#### 步骤2: 前端渲染

```vue
<el-select v-model="formData.leave_type">
  <el-option label="年假" value="annual" />
  <el-option label="病假" value="sick" />
  <el-option label="事假" value="personal" />
</el-select>
```

**用户看到：** "年假"、"病假"、"事假"  
**实际值：** `annual`、`sick`、`personal`

#### 步骤3: 用户选择

用户选择"年假" → `formData.leave_type = "annual"`

#### 步骤4: 提交表单

```json
{
  "leave_type": "annual"  // 提交的是编码
}
```

#### 步骤5: 数据存储

```sql
INSERT INTO form_history (field_code, field_value)
VALUES ('leave_type', 'annual');  -- 存储的是编码
```

#### 步骤6: 推荐查询

```sql
SELECT field_value, COUNT(*) 
FROM form_history 
WHERE field_code = 'leave_type'
GROUP BY field_value;

-- 结果：
-- annual: 100次
-- sick: 50次
```

#### 步骤7: 推荐返回

```json
{
  "recommendations": [
    {
      "value": "annual",      // 编码
      "label": "年假",         // 中文
      "score": 0.95,
      "source": "db"
    },
    {
      "value": "sick",
      "label": "病假",
      "score": 0.85,
      "source": "db"
    }
  ]
}
```

#### 步骤8: 前端显示推荐

```vue
<el-option label="年假" value="annual" />
<el-option label="病假" value="sick" />
```

## 📝 注意事项

### 1. 本体必须配置options

确保本体文件中定义了字段的枚举值和标签：

```json
{
  "fieldCode": "leave_type",
  "type": "enum",
  "options": [
    {"value": "annual", "label": "年假"},
    {"value": "sick", "label": "病假"}
  ]
}
```

### 2. 向后兼容

- ✅ 旧的纯字符串格式仍然支持
- ✅ 新的{value, label}格式优先
- ✅ 平滑过渡，无需数据迁移

### 3. Schema生成

AI生成表单Schema时，应该从本体读取options：

```python
# 伪代码

    {"value": opt.value, "label": opt.label}

]
```
