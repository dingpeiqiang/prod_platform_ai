**涉及文件：**
- `frontend/src/App.vue`

**代码变更：**
```javascript
// 修改前
const onTitleUpdate = (id, title) => {
  const s = sessions.value.find(s => s.id === id)
  if (s) { s.title = title; s.updatedAt = Date.now(); saveSessions() }
}

// 修改后
const onTitleUpdate = async (id, title) => {
  const s = sessions.value.find(s => s.id === id)
  if (s) {
    s.title = title
    s.updatedAt = Date.now()
    saveSessions()
    // 同步到后端数据库
    if (s.dbSessionId) {
      await apiUpdateSessionTitle(s.dbSessionId, title)
    }
  }
}
```

### 3. ✅ 优化会话匹配机制

**问题描述：**
- 使用时间戳匹配本地和数据库会话（误差5秒）
- 在并发或网络延迟情况下容易出错
- 匹配可靠性低

**解决方案：**
- 采用更可靠的匹配策略：优先使用已存储的dbSessionId
- 无dbSessionId时，通过标题+创建时间（30秒窗口）进行匹配
- 添加详细的日志输出，便于调试和问题排查

**涉及文件：**
- `frontend/src/App.vue`

**代码变更：**
```javascript
// 修改前：仅使用时间戳匹配（5秒窗口）
const matched = dbSessions.find(d =>
  Math.abs(new Date(d.created_at).getTime() - s.createdAt) < 5000
)

// 修改后：标题 + 时间戳组合匹配（30秒窗口）
const matched = dbSessions.find(d => {
  // 优先匹配标题相同的会话
  if (s.title && d.title && s.title === d.title) {
    // 如果标题相同，再检查创建时间是否在合理范围内（30秒）
    const timeDiff = Math.abs(new Date(d.created_at).getTime() - s.createdAt)
    return timeDiff < 30000
  }
  return false
})
```

**优势：**
- 标题匹配更可靠，避免时间戳误差问题
- 30秒窗口更宽容，适应网络延迟场景
- 添加日志输出，便于问题追踪

### 4. ✅ 实现批量消息保存

**问题描述：**
- 前端saveMessages函数逐个保存消息
- 每个消息都需要单独的HTTP请求
- 大量消息时性能低下

**解决方案：**
- 后端添加批量保存API端点 `/sessions/{session_id}/messages/batch`
- 前端优化saveMessages函数，消息数>3时使用批量API
- 批量失败时自动降级为逐个保存，保证可靠性

**涉及文件：**
- `backend/app/api/chat_v2.py`
- `frontend/src/services/chatApi.js`

**代码变更：**

后端新增批量API：
```python
@router.post("/sessions/{session_id}/messages/batch", response_model=Dict[str, Any])
async def create_messages_batch(
    session_id: str,
    request: BatchMessageCreateRequest,
    db: Session = Depends(get_db)
):
    """批量保存消息（提高性能）"""
    results = []
    for msg_req in request.messages:
        result = ChatServiceV2.save_message(...)
        if result:
            results.append(result)
        else:
            raise HTTPException(500, detail=f"批量保存失败，已保存 {len(results)} 条消息")
    
    logger.info("[chat_v2] 批量保存消息 session=%s count=%d", session_id, len(results))
    return {"success": True, "count": len(results), "results": results}
```

前端优化批量保存：
```javascript
export async function saveMessages(sessionId, messages) {
  const done = messages.filter(m => m.done !== false)
  
  // 如果消息数量较多，使用批量 API
  if (done.length > 3) {
    try {
      const batchData = done.map(msg => {...})
      const resp = await fetch(`${BASE}/sessions/${encodeURIComponent(sessionId)}/messages/batch`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ messages: batchData })
      })
      
      if (!resp.ok) {
        // 批量失败时降级为逐个保存
        for (const msg of done) {
          await saveMessage(sessionId, msg)
        }
      }
    } catch (e) {
      // 出错时降级为逐个保存
      for (const msg of done) {
        await saveMessage(sessionId, msg)
      }
    }
  } else {
    // 消息数量少时直接逐个保存
    for (const msg of done) {
      await saveMessage(sessionId, msg)
    }
  }
}
```

**性能提升：**
- 减少HTTP请求次数（从N次减少到1次）
- 降低网络开销和延迟
- 提升用户体验，特别是历史消息加载场景

### 5. ✅ 优化元数据存储

**问题描述：**
- ChatMessageMetadata表使用条件判断进行序列化
- 不同类型数据采用不同序列化方式（JSON vs String）
- 读取时需要额外处理，逻辑复杂

**解决方案：**
- 统一使用JSON序列化所有metadata值
- 写入时：所有值都通过json.dumps序列化
- 读取时：统一通过json.loads反序列化，恢复原始数据类型
- 添加异常处理，兼容非JSON格式的旧数据

**涉及文件：**
- `backend/app/services/chat_service_v2.py`

**代码变更：**

写入优化：
```python
# 修改前
value=json.dumps(value) if isinstance(value, (dict, list)) else str(value) if value is not None else None

# 修改后
serialized_value = json.dumps(value, ensure_ascii=False) if value is not None else None
```

读取优化：
```python
# 修改前
result["metadata"] = {x.meta_key: x.value for x in m.msg_metadata}

# 修改后
metadata_dict = {}
for x in m.msg_metadata:
    try:
        # 尝试解析 JSON，如果失败则保持原字符串
        metadata_dict[x.meta_key] = json.loads(x.value) if x.value is not None else None
    except (json.JSONDecodeError, TypeError):
        # 如果不是有效的 JSON，直接返回字符串
        metadata_dict[x.meta_key] = x.value
result["metadata"] = metadata_dict
```

**优势：**
- 统一序列化逻辑，简化代码
- 保持数据类型完整性（数字、布尔值、对象等）
- 向后兼容，支持旧格式数据
- 使用ensure_ascii=False，支持中文等非ASCII字符
