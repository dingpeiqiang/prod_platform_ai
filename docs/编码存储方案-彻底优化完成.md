## 🎯 问题发现

之前的编码存储方案**没有优化彻底**，存在以下遗漏：

### ❌ 问题点

1. **recommendation_engine_v4.py** - 推荐引擎返回的数据只有`value`（编码），没有`label`（中文）
2. **form_service.py** - 表单服务构建recommend数据时也没有添加`label`字段
3. **前端显示** - 由于后端没返回label，前端直接显示编码（如`annual`而不是`年假`）

```python
def _build_enum_recommendations(self, field_code: str, enum_values: List[str]) -> List[Dict]:
    """
    构建枚举值推荐列表
    ⚠️ 重要：返回 {value, label} 格式，value是编码，label是中文
    """
    from app.services.ontology_service import OntologyService
    
    recommendations = []
    for i, value in enumerate(enum_values):
        # 尝试从本体获取label
        label = value  # 默认使用value本身
        try:
            if form_code:
                ontology_result = OntologyService.get_form_constraint(form_code)
                if ontology_result.get('success'):
                    ontology = ontology_result.get('constraints', {})
                    entities = ontology.get('entities', [])
                    for entity in entities:
                        fields = entity.get('fields', [])
                        for field_def in fields:
                            if field_def.get('fieldCode') == field_code:
                                options = field_def.get('options', [])
                                for opt in options:
                                    if isinstance(opt, dict) and opt.get('value') == value:
                                        label = opt.get('label', value)
                                        break
        except Exception as e:
            logger.warning(f"[V4Engine] 获取字段标签失败: {e}")
        
        recommendations.append({
            "value": value,
            "label": label,  # ✅ 新增：中文标签
            "field_code": field_code,
            ...
        })
    
    return recommendations
```

#### 1.3 新增_get_field_label辅助方法

```python
def _get_field_label(self, form_code: str, field_code: str, code: str) -> str:
    """
    从本体定义中获取字段的中文标签
    
    Args:
        form_code: 表单编码
        field_code: 字段编码
        code: 字段值（编码）
    
    Returns:
        中文标签，如果没有映射则返回原值
    """
    try:
        from app.services.ontology_service import OntologyService
        
        ontology_result = OntologyService.get_form_constraint(form_code)
        if not ontology_result.get('success'):
            return code
        
        ontology = ontology_result.get('constraints', {})
        entities = ontology.get('entities', [])
        
        for entity in entities:
            fields = entity.get('fields', [])
            for field_def in fields:
                if field_def.get('fieldCode') == field_code:
                    options = field_def.get('options', [])
                    for opt in options:
                        if isinstance(opt, str):
                            if opt == code:
                                return opt
                        elif isinstance(opt, dict):
                            if opt.get('value') == code:
                                return opt.get('label', code)
                    return code
        
        return code
    except Exception as e:
        logger.warning(f"[V4Engine] 获取字段标签失败: {e}")
        return code
```

## 📊 完整数据流

### 修复后的完整流程

```
┌─────────────────────────┐
│  用户导入数据            │
│  {"leave_type":"annual"} │  ← 编码
└───────────┬─────────────┘
            ↓
┌─────────────────────────┐
│  存储到数据库            │
│  FormHistory             │
│  field_value: "annual"   │  ← 存储编码
└───────────┬─────────────┘
            ↓
┌─────────────────────────┐
│  推荐引擎查询            │
│  SELECT field_value      │
│  FROM form_history       │
│  WHERE field_code='...'  │
│  → 结果: "annual"        │  ← 查询到编码
└───────────┬─────────────┘
            ↓
┌─────────────────────────┐
│  获取中文标签            │
│  _get_field_label()      │
│  annual → 年假           │  ← 从本体映射
└───────────┬─────────────┘
            ↓
┌─────────────────────────┐
│  返回推荐数据            │
│  {                       │
│    "value": "annual",    │  ← 用于提交
│    "label": "年假"       │  ← 用于显示
│  }                       │
└───────────┬─────────────┘
            ↓
┌─────────────────────────┐
│  前端渲染                │
│  显示: [年假]            │  ← 用户看到中文
│  点击 → value="annual"   │  ← 提交编码
└─────────────────────────┘
```

## 📝 注意事项

### 1. 本体配置

确保所有枚举字段都在本体中配置了options：

```json
{
  "fieldCode": "leave_type",
  "fieldName": "请假类型",
  "fieldType": "select",
  "options": [
    {"value": "annual", "label": "年假"},
    {"value": "sick", "label": "病假"},
    {"value": "personal", "label": "事假"}
  ]
}
```

### 2. 向后兼容

代码支持两种options格式：
- **字符串数组**: `["年假", "病假"]` - 旧格式
- **对象数组**: `[{"value": "annual", "label": "年假"}]` - 新格式（推荐）

### 3. 容错处理

如果本体中没有配置options，系统会：
- 返回编码作为label
- 记录警告日志
- 不影响正常使用
