# 自动化校验能力实现方案

## 概述

自动化校验能力完全基于**本体配置**实现，支持：
- **内置校验规则**（开箱即用）
- **自定义校验规则**（脚本化）

---

## 核心架构

```
┌─────────────────────────────────────────────────┐
│              校验引擎 (ValidationEngine)       │
│  ┌───────────────────────────────────────┐   │
│  │ 内置规则 │ 自定义规则注册器             │   │
│  │ minLength, maxLength, min, max,     │   │
│  │ pattern, email, phone, idCard, url,   │   │
│  │ enum, dateMin, dateMax, custom            │   │
│  └───────────────────────────────────────┘   │
│         ↓                     ↓                 │
│    单字段校验         表单整体校验               │
└─────────────────────────────────────────────────┘
```

---

## 内置校验规则（12种）

| 规则类型 | 说明 | 示例值 |
|---------|------|---------|
| `minLength` | 最小长度 | 3 |
| `maxLength` | 最大长度 | 20 |
| `min` / `minimum` | 最小值 | 18 |
| `max` / `maximum` | 最大值 | 100 |
| `pattern` | 正则表达式 | `^1[3-9]\\d{9}$` |
| `email` | 邮箱格式 | null（无需值） |
| `phone` | 手机号 | null（无需值） |
| `idCard` | 身份证号 | null（无需值） |
| `url` | URL格式 | null（无需值） |
| `enum` | 枚举值 | `["技术部", "产品部"]` |
| `dateMin` | 日期最小值 | `"2024-01-01"` |
| `dateMax` | 日期最大值 | `"2025-12-31"` |
| `custom` | 自定义脚本 | 见下方说明 |

```json
{
  "fieldCode": "username",
  "fieldName": "用户名",
  "fieldType": "input",
```

## 自定义校验规则（复杂校验支持）

### 方式1：Simple简单条件式

使用简单的表达式，`value` 变量表示字段值

```json
{
  "fieldCode": "evenNumber",
  "fieldName": "偶数",
  "fieldType": "number",
  "required": true,
  "rules": [
    {
      "rule_type": "custom",
      "rule_value": {
        "type": "simple",
        "condition": "value % 2 == 0"
      },
      "message": "请输入偶数"
    }
  ]
}
```

### 方式2：Lambda表达式

使用完整的Python lambda 函数

```json
{
  "fieldCode": "strongPassword",
  "fieldName": "强密码",
  "fieldType": "input",
  "required": true,
  "rules": [
    {
      "rule_type": "custom",
      "rule_value": {
        "type": "lambda",
        "code": "lambda x: len(x) >= 12 and any(c.isupper() for c in x) and any(c.islower() for c in x) and any(c.isdigit() for c in x)"
      },
      "message": "密码需至少12位，包含大小写字母和数字"
    }
  ]
}
```

### 方式3：代码注册复杂规则（需修改代码）

对于非常复杂的校验，可以在代码中注册：

```python
from app.services.validation_service import validation_engine

def validate_credit_card(value, rule_value):
    # Luhn算法校验信用卡号
    def luhn_check(card_number):
        digits = [int(d) for d in str(card_number) if d.isdigit()]
        odd_sum = sum(digits[-1::-2])
        even_sum = sum(sum(divmod(2 * d, 10)) for d in digits[-2::-2])
        return (odd_sum + even_sum) % 10 == 0
    return luhn_check(value)

# 注册自定义规则
validation_engine.register_custom_rule("creditCard", validate_credit_card)
```

然后在本体中使用：

```json
{
  "rule_type": "creditCard",
  "rule_value": null,
  "message": "信用卡号格式不正确"
}
```

## 校验API

### 1. 单字段校验

**接口：** `POST /api/v1/validation/field`

**请求：**
```json
{
  "fieldValue": "test@example.com",
  "rules": [
    {
      "rule_type": "email",
      "rule_value": null,
      "message": "请输入有效的邮箱"
    }
  ]
}
```

**响应：**
```json
{
  "success": true,
  "valid": true,
  "errors": []
}
```

### 2. 表单整体校验

**接口：** `POST /api/v1/validation/form`

**请求：**
```json
{
  "data": {
    "username": "test",
    "email": "invalid"
  },
  "fields": [
    {
      "fieldCode": "username",
      "fieldName": "用户名",
      "required": true,
      "rules": [
        {"rule_type": "minLength", "rule_value": 3, "message": "用户名至少3个字符"}
      ]
    },
    {
      "fieldCode": "email",
      "fieldName": "邮箱",
      "required": true,
      "rules": [
        {"rule_type": "email", "rule_value": null, "message": "请输入有效的邮箱"}
      ]
    }
  ]
}
```

**响应：**
```json
{
  "success": true,
  "valid": false,
  "errors": ["请输入有效的邮箱"]
}
```

### 3. 表单提交自动校验

**接口：** `POST /api/v1/form/submit`

提交时会自动校验，校验失败直接返回错误：

**失败响应示例：
```json
{
  "success": false,
  "message": "请输入有效的邮箱; 年龄不能小于18岁",
  "formInstanceId": null
}
```

## 总结

| 能力 | 实现方式 |
|------|---------|
| 基础校验 | 内置规则，本体配置 |
| 格式校验 | 内置规则，无需值 |
| 自定义简单校验 | custom: simple表达式 |
| 自定义复杂校验 | custom: lambda函数 |
| 超复杂校验 | 代码注册 |
