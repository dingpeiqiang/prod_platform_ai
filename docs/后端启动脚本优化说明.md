# 后端启动脚本优化说明

## 🎯 优化目标

简化后端启动流程，默认直接启动生产模式，无需用户交互。

---

## ✅ 优化内容

### 1. 移除交互式选择

**优化前**:
```batch
echo [4/4] 选择启动模式：
echo.
echo    [1] 生产模式（uvicorn，无 reload，日志完整）
echo    [2] 开发模式（uvicorn --reload，改代码自动重启）
echo.
set /p MODE=请输入选项 [1/2，默认1]:

if "%MODE%"=="2" goto DEV
```

**优化后**:
```batch
echo [4/4] 检查启动参数...
if "%1"=="dev" goto DEV
if "%1"=="--dev" goto DEV
if "%1"=="-d" goto DEV

echo.
echo [PROD] 启动生产模式（默认）...
echo    - 如需开发模式，请使用: start-backend.bat dev
```

---

### 2. 支持命令行参数

现在可以通过命令行参数选择启动模式：

#### 生产模式（默认）
```bash
start-backend.bat
```
或显式指定：
```bash
start-backend.bat prod
```

#### 开发模式
```bash
start-backend.bat dev
```
或：
```bash
start-backend.bat --dev
start-backend.bat -d
```

## 🚀 使用场景

### 场景1: 日常开发（推荐）

直接双击 `start-backend.bat` 或运行：
```bash
start-backend.bat
```

系统会自动启动生产模式，快速进入开发状态。

---

### 场景2: 需要热重载

修改代码时希望自动重启，使用：
```bash
start-backend.bat dev
```

这会启用 `--reload` 参数，文件修改后自动重启服务。

---

### 场景3: 自动化脚本

在CI/CD或其他脚本中调用：
```bash
# 启动后端
start start-backend.bat

# 等待服务启动
timeout /t 5

# 执行测试
curl http://localhost:6173/health
```

## 💡 技术细节

### 参数解析逻辑

```batch
if "%1"=="dev" goto DEV
if "%1"=="--dev" goto DEV
if "%1"=="-d" goto DEV
```

- `%1`: 第一个命令行参数
- 支持三种别名：`dev`, `--dev`, `-d`
- 未提供参数时，默认执行生产模式

### 端口检测

脚本会自动检测端口占用：
- 优先使用 6173 端口
- 如果被占用，自动切换到 6174 端口
- 避免端口冲突导致的启动失败

## 🔧 相关脚本

### start-all.bat

同时启动前后端：
```bash
start-all.bat
```

这个脚本会：
1. 启动后端（生产模式）
2. 启动前端
3. 打开浏览器访问应用

---

## 📝 注意事项

1. **首次运行**: 会自动创建虚拟环境并安装依赖，可能需要几分钟
2. **端口占用**: 如果6173和6174都被占用，需要手动关闭占用进程
3. **日志查看**: 
   - 终端实时显示
   - 完整日志保存在 `backend/app/logs/app.log`
