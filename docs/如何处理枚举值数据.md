# 📋 处理枚举值数据 - 完整指南

## 🎯 问题场景

在真实业务系统中，数据库存储的往往是**枚举值**而非中文显示：

### 示例对比

| 字段 | 数据库存储（枚举值） | 用户看到（显示值） |
|------|-------------------|------------------|
| leave_type | `annual` | 年假 |
| leave_type | `sick` | 病假 |
| department | `tech` | 技术部 |
| department | `sales` | 销售部 |

---

## ✅ 解决方案

### 方案1：使用Schema的valueTransform（推荐）

#### 步骤1: 创建schema.json文件

```json
{
  "formCode": "leave",
  "formName": "请假申请",
  "description": "OA系统导出的请假记录",
  
  "valueTransform": {
    "leave_type": {
      "annual": "年假",
      "sick": "病假",
      "personal": "事假"
    },
    "department": {
      "tech": "技术部",
      "sales": "销售部",
      "hr": "人事部"
    }
  },
  
  "userField": "applicant_name"
}
```

#### 步骤2: 准备数据文件（使用枚举值）

```jsonl
{"applicant_name":"张三","department":"tech","leave_type":"annual","days":5}
{"applicant_name":"李四","department":"sales","leave_type":"personal","days":3}
{"applicant_name":"王五","department":"tech","leave_type":"sick","days":2}
```

#### 步骤3: 执行导入

系统会**自动转换**：
- `tech` → `技术部`
- `annual` → `年假`
- `sales` → `销售部`

#### 步骤4: 查看结果

导入报告会显示**转换后的中文值**：

```
🔍 字段分布 (Top 5):

▼ department (2 种不同值):
  技术部: 2 次
  销售部: 1 次

▼ leave_type (3 种不同值):
  年假: 1 次
  事假: 1 次
  病假: 1 次
```

### 方案2：在导出时直接转换

如果你的业务系统支持，可以在导出数据时就转换为中文：

#### SQL示例

```sql
SELECT 
  applicant_name,
  CASE department
    WHEN 'tech' THEN '技术部'
    WHEN 'sales' THEN '销售部'
    WHEN 'hr' THEN '人事部'
  END as department,
  CASE leave_type
    WHEN 'annual' THEN '年假'
    WHEN 'sick' THEN '病假'
    WHEN 'personal' THEN '事假'
  END as leave_type,
  start_date,
  end_date,
  days
FROM leave_records
```

#### Python示例

```python
import csv
import json

# 定义映射关系
DEPARTMENT_MAP = {
    'tech': '技术部',
    'sales': '销售部',
    'hr': '人事部'
}

LEAVE_TYPE_MAP = {
    'annual': '年假',
    'sick': '病假',
    'personal': '事假'
}

with open('leave_export.csv', 'r', encoding='utf-8-sig') as csvfile:
    reader = csv.DictReader(csvfile)
    with open('my_leave.data.jsonl', 'w', encoding='utf-8') as jsonl:
        for row in reader:
            record = {
                "applicant_name": row["申请人"],
                "department": DEPARTMENT_MAP.get(row["部门"], row["部门"]),
                "leave_type": LEAVE_TYPE_MAP.get(row["请假类型"], row["请假类型"]),
                "start_date": row["开始日期"],
                "end_date": row["结束日期"],
                "days": int(row["天数"])
            }
            jsonl.write(json.dumps(record, ensure_ascii=False) + '\n')
```

### 方案3：混合方式（部分转换）

如果只有部分字段需要转换：

```json
{
  "formCode": "leave",
  "valueTransform": {
    "leave_type": {
      "annual": "年假",
      "sick": "病假"
    }
    // department 不配置，保持原值
  }
}
```

这样：
- `leave_type` 会转换：`annual` → `年假`
- `department` 保持原值：`tech` → `tech`

## ⚠️ 注意事项

### 1. 转换是单向的

```
枚举值 → 中文显示值
annual → 年假
```

**不能反向转换**，所以请确保：
- Schema中的映射是完整的
- 所有可能的枚举值都有对应的中文

### 2. 未配置的字段保持原值

如果某个字段没有在valueTransform中配置：
- 枚举值会保持原样存储
- 推荐时会显示枚举值而非中文

### 3. 大小写敏感

```json
{
  "leave_type": {
    "annual": "年假",    // ✅ 匹配 "annual"
    "Annual": "年假",    // ❌ 不会匹配 "annual"
    "ANNUAL": "年假"     // ❌ 不会匹配 "annual"
  }
}
```

确保映射的key与数据中的值完全一致。

### 4. 特殊字符

如果枚举值包含特殊字符，需要正确转义：

```json
{
  "status": {
    "pending_review": "待审核",
    "approved_v2": "已批准V2"
  }
}
```

## 💡 最佳实践

### 1. 优先使用valueTransform

✅ **推荐做法：**
- 数据保持原始的枚举值
- 在schema中配置valueTransform
- 导入时自动转换

❌ **不推荐：**
- 手动转换数据后再导入
- 容易出错且难以维护

### 2. 保持映射的一致性

- 同一个字段的映射在所有schema中保持一致
- 建立统一的枚举值字典

### 3. 文档化映射关系

在schema中添加注释：

```json
{
  "formCode": "leave",
  "_comment": "枚举值映射参考OA系统字典表",
  "valueTransform": {
    "leave_type": {
      "annual": "年假",  // OA系统代码: A
      "sick": "病假",    // OA系统代码: S
      "personal": "事假" // OA系统代码: P
    }
  }
}
```

### 4. 定期更新映射

当业务系统增加新的枚举值时：
1. 更新schema中的valueTransform
2. 重新导入新数据
3. 验证转换是否正确

## 📞 常见问题

### Q1: 如果枚举值很多怎么办？

**A:** 可以：
1. 只转换常用的枚举值
2. 其他的保持原值
3. 或者在导出时就转换好

### Q2: 转换会影响性能吗？

**A:** 不会。转换是在导入时一次性完成的，不影响查询性能。

### Q3: 能否动态加载映射关系？

**A:** 当前实现是从schema.json静态加载。如果需要动态加载，可以从数据库或配置文件读取。

### Q4: 如果数据中既有枚举值又有中文怎么办？

**A:** valueTransform只会转换匹配的枚举值：
- `"annual"` → `"年假"` （转换）
- `"年假"` → `"年假"` （保持不变，因为没有匹配到key）

## 🎯 总结

处理枚举值的核心要点：

1. ✅ **使用schema的valueTransform配置映射**
2. ✅ **数据文件保持原始的枚举值**
3. ✅ **导入时自动转换为中文显示值**
4. ✅ **推荐引擎使用转换后的值**
5. ✅ **用户看到的是友好的中文**

**这样既保持了数据的规范性，又提供了良好的用户体验！** 🎉
