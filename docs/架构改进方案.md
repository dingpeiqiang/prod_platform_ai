# 架构改进方案

## 当前问题分析

1. **硬编码严重** - 本体、推荐数据、场景映射全部硬编码在代码中
2. **AI能力缺失** - 仅使用规则匹配，无真实LLM能力
3. **配置化不足** - 缺少动态配置管理
4. **扩展性差** - 新增场景需要修改代码

## 改进目标

1. ✅ **全配置化** - 本体、场景、规则全部可配置
2. ✅ **真实AI能力** - 集成LLM进行智能解析
3. ✅ **动态管理** - 支持运行时配置更新
4. ✅ **通用框架** - 提供灵活的扩展机制

## 改进内容

### 1. 配置化架构

#### 1.1 配置目录结构
```
config/
├── ontologies/              # 本体定义（JSON/YAML）
│   ├── sales_order.json
│   ├── leave.json
│   └── expense.json
├── scenes/                  # 场景配置
│   └── scene_mapping.json
├── templates/               # 表单模板
│   └── default_form.json
├── prompts/                 # LLM提示词
│   ├── intent_recognition.txt
│   └── field_extraction.txt
└── app_config.json          # 应用配置
```
