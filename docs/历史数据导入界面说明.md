## 🎯 导入流程（3步向导）

### 步骤1: 选择表单类型

![步骤1](./images/import-step1.png)

**界面元素：**
- **步骤指示器**：显示当前在第几步（共3步）
- **表单列表表格**：
  - 单选框：选择要导入的表单
  - 表单编码：如 `leave`、`sales_order`
  - 表单名称：如 "请假申请"、"销售订单"
  - 数据格式：JSONL 或 CSV
  - 描述：数据来源说明
  - Schema：是否配置了元数据
- **表单信息卡片**：显示选中表单的详细信息
- **下一步按钮**：进入配置页面

**操作：**
1. 从列表中选择一个表单类型
2. 查看右侧的表单详细信息
3. 点击"下一步"

---

### 步骤2: 确认导入配置

![步骤2](./images/import-step2.png)

**界面元素：**
- **导入配置卡片**：
  - 表单类型：显示选中的表单
  - 数据源：显示数据文件路径
  - 限制条数：可选，用于测试导入
- **注意事项提示**：
  - ⚠️ 导入可能需要几分钟
  - ⚠️ 重复导入会追加数据
  - ⚠️ 导入后可用于智能推荐
- **上一步/开始导入按钮**

**操作：**
1. 确认表单类型和数据源
2. （可选）设置限制条数，如输入 `10` 仅导入前10条
3. 阅读注意事项
4. 点击"开始导入"

## 💡 使用技巧

### 1. 预览模式

在命令行中使用 `--dry-run` 参数可以预览导入效果而不写入数据库：

```bash
python -m app.scripts.import_history leave --dry-run
```

这会显示：
- 总记录数
- 数据格式（扁平/嵌套）
- 顶层字段值分布
- 嵌套子字段值分布
- 首条记录样例

### 2. 限制条数测试

首次导入时，建议先限制条数进行测试：

**Web界面：**
- 在步骤2的"限制条数"中输入 `5` 或 `10`

**命令行：**
```bash
python -m app.scripts.import_history leave --limit 10
```

### 3. 批量导入

如果需要导入多个表单类型：

**命令行方式：**
```bash
# 导入全部
python -m app.scripts.import_history --all

# 逐个导入
python -m app.scripts.import_history leave
python -m app.scripts.import_history sales_order
```

**Web界面：**
- 重复导入流程，每次选择一个表单类型

## 🔗 相关功能

导入完成后，你可以在以下场景看到效果：

### 1. 聊天窗口智能推荐

当你填写表单时，系统会基于导入的历史数据推荐值：

```
用户: "我要请假"
AI: [生成请假表单]

表单字段：
- 申请人: [下拉推荐: 张三, 李四, 王五...]
- 部门:   [下拉推荐: 技术部, 销售部, 人事部...]
- 请假类型: [下拉推荐: 年假, 病假, 事假...]
```

### 2. 动态默认值

某些字段会根据历史记录自动填充默认值：

```
- 如果张三经常请年假，当他再次请假时：
  请假类型 自动推荐为 "年假"
```

### 3. 数据分析

通过字段分布统计，你可以了解：
- 哪些值最常用
- 数据的多样性
- 潜在的异常值

## 📝 注意事项

1. **数据准备**：确保数据文件格式正确
2. **字段匹配**：字段名必须与本体定义一致
3. **避免重复**：重复导入会导致数据重复
4. **耐心等待**：大数据量导入需要时间
5. **查看日志**：遇到问题查看 `backend/logs/app.log`
