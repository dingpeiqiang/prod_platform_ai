# 📥 历史数据导入 - 完整流程说明

## 🎯 正确的用户流程

### 流程概览

```
用户在聊天窗口表达导入意图
    ↓
AI识别意图，返回导入入口
    ↓
前端显示DataImportEntry组件
    ↓
用户下载模板文件
    ↓
用户准备自己的真实业务数据
    ↓
用户上传JSONL文件
    ↓
系统执行导入
    ↓
显示导入结果报告
```

---

## 📝 详细步骤

### 步骤1: 用户表达导入意图

用户在聊天窗口输入：
- "我要导入历史数据"
- "我想导入请假记录"
- "帮我导入销售订单数据"

### 步骤2: AI识别意图

后端Intent识别器识别为 `manage_history` 意图，action为 `import`。

**ManageHistoryHandler返回：**
```json
{
  "type": "import_entry",
  "formCode": "leave",
  "formName": "请假申请",
  "message": "请上传「请假申请」的历史数据文件（JSONL格式）",
  "template_url": "/api/v1/config/import/template/leave",
  "upload_url": "/api/v1/config/import/upload"
}
```

### 步骤3: 前端显示导入入口

HistoryPanel检测到 `historyData.type === 'import_entry'`，渲染DataImportEntry组件。

**界面显示：**
- 📋 步骤1: 下载导入模板
- ✏️ 步骤2: 准备您的数据
- 📤 步骤3: 上传并导入

### 步骤4: 用户下载模板

用户点击"下载模板文件"按钮：

**后端API：**
```
GET /api/v1/config/import/template/{formCode}
```

**响应：**
- 从 `import_data_examples/` 目录返回示例文件
- 如果没有示例文件，返回一个简单的模板

**用户获得：**
- `leave_template.jsonl` 文件
- 了解正确的JSONL格式

### 步骤5: 用户准备数据

用户按照以下步骤准备数据：

1. **从业务系统导出数据**
   - OA系统 → 导出请假记录（CSV/Excel）
   - ERP系统 → 导出订单数据
   - CRM系统 → 导出客户信息

2. **转换为JSONL格式**

   使用Python脚本：
   ```python
   import csv
   import json
   
   with open('leave_export.csv', 'r', encoding='utf-8-sig') as csvfile:
       reader = csv.DictReader(csvfile)
       with open('my_leave.data.jsonl', 'w', encoding='utf-8') as jsonl:
           for row in reader:
               record = {
                   "applicant_name": row["申请人"],
                   "department": row["部门"],
                   "leave_type": row["请假类型"],
                   "start_date": row["开始日期"],
                   "end_date": row["结束日期"],
                   "days": int(row["天数"])
               }
               jsonl.write(json.dumps(record, ensure_ascii=False) + '\n')
   ```

3. **验证格式**
   - 每行一个有效的JSON对象
   - 字段名与本体定义一致
   - UTF-8编码

### 步骤6: 用户上传文件

用户在DataImportEntry组件中：
1. 拖拽或点击选择文件
2. 选择准备好的 `.jsonl` 文件
3. 点击"开始导入"按钮

**前端发送请求：**
```javascript
POST /api/v1/config/import/upload
Content-Type: multipart/form-data

FormData:
  - file: my_leave.data.jsonl
  - formCode: leave
```

### 步骤7: 后端执行导入

**upload_and_import API处理流程：**

1. **接收文件**
   - 验证文件扩展名（必须是.jsonl）
   - 保存到临时目录

2. **执行导入**
   - 调用 `import_form_data()` 函数
   - 读取JSONL文件
   - 扁平化嵌套结构
   - 值转换（如果有schema）
   - 写入数据库（FormInstance + FormHistory）

3. **生成统计**
   - 计算字段值分布
   - 提取Top 5值
   - 检测嵌套字段

4. **清理临时文件**
   - 删除临时目录和文件

**返回响应：**
```json
{
  "success": true,
  "message": "成功导入 25 条「leave」历史数据",
  "formCode": "leave",
  "totalImported": 25,
  "totalSource": 25,
  "totalSkipped": 0,
  "totalErrors": 0,
  "fieldStats": [
    {
      "fieldCode": "applicant_name",
      "distinctValues": 8,
      "topValues": [
        {"value": "张三", "count": 5},
        {"value": "李四", "count": 4},
        {"value": "王五", "count": 3}
      ]
    },
    {
      "fieldCode": "department",
      "distinctValues": 5,
      "topValues": [
        {"value": "技术部", "count": 10},
        {"value": "销售部", "count": 6}
      ]
    }
  ],
  "nestedFields": []
}
```

### 步骤8: 显示导入结果报告

前端收到响应后：
1. 关闭进度条
2. 显示结果对话框
3. 展示详细的导入报告

**报告显示内容：**
- ✅ 导入成功/失败状态
- 📊 数据统计（源记录、实际导入、跳过、错误）
- 🔍 字段分布（Top 5值和出现次数）
- ℹ️ 嵌套字段处理说明

## ⚠️ 重要说明

### 1. 不提供虚拟数据
- ❌ 系统不会预置任何示例数据
- ✅ 用户必须从自己的业务系统导出真实数据
- ✅ 示例文件仅作为格式参考

### 2. 数据格式要求
- ✅ 文件格式：.jsonl
- ✅ 每行一个有效的JSON对象
- ✅ 字段名与本体定义一致
- ✅ UTF-8编码

### 3. 数据安全
- ✅ 上传的文件保存到临时目录
- ✅ 导入完成后立即清理
- ✅ 不会保留用户上传的原始文件

### 4. 错误处理
- ✅ 文件格式验证
- ✅ 导入失败时显示错误信息
- ✅ 详细的日志记录
