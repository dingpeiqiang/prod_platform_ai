# 豆包式会话管理逻辑说明

## 核心设计理念（参考豆包）

**左侧栏只显示有内容的会话，空会话不会出现在左侧栏中。**

## 完整工作流程

### 1. 初始状态（用户首次登录）

```
用户登录
↓
initDbSessions() 加载会话
↓
如果没有会话 → activeSessionId = '', activeDbSessionId = ''
↓
显示聊天欢迎界面
↓
左侧栏为空（没有任何会话）
```

### 2. 用户发送第一条消息

```
用户在输入框输入："帮我填写销售订单"
↓
点击发送
↓
ChatAssistant.vue 检测到 props.sessionId 为空
↓
生成临时 sessionId: sess_xxx
↓
调用 ensureDbSession(tempSessionId, "帮我填写销售订单")
↓
创建数据库会话（标题="帮我填写销售订单"）
↓
emit('session-init', { localId: tempSessionId, dbSessionId: xxx })
↓
App.vue 的 onSessionInit 接收回调
↓
创建本地会话对象，dbSessionId 已设置
↓
添加到 sessions.value（左侧栏显示）
↓
标题通过 title-update 事件设置为"帮我填写销售订单"
↓
左侧栏显示新会话："帮我填写销售订单" ✅
```

### 3. 点击"新建对话"按钮

```
用户点击"新建对话"
↓
onNewSession() 被调用
↓
清除当前会话的 localStorage 消息数据
↓
ChatAssistant 重新加载（显示空的消息列表）
↓
不创建任何会话记录
↓
左侧栏不变（仍然只显示有内容的会话）
↓
用户输入新消息并发送
↓
在同一个会话中继续对话（不会创建新会话）
```

**注意**：这与豆包的行为一致 - "新建对话"只是清空聊天窗口，不创建新会话。

### 4. 切换会话

```
用户点击左侧栏的某个会话
↓
onSwitchSession(sessionId)
↓
activeSessionId = sessionId
↓
activeDbSessionId = session.dbSessionId
↓
ChatAssistant 从数据库加载该会话的历史消息
↓
显示完整的对话历史
```

### 5. 删除会话

```
用户点击会话的删除按钮
↓
deleteSession(sessionId)
↓
如果会话有 dbSessionId，调用 API 删除数据库会话
↓
从 sessions.value 中移除
↓
清除 localStorage 数据
↓
如果删除的是当前激活的会话，切换到其他会话
```

## 关键代码实现

### App.vue - onNewSession（新建对话按钮）

```javascript
// 参考豆包逻辑：点击"新建对话"只是清空当前聊天窗口，不创建新会话
const onNewSession = () => {
  console.log('[App] 清空当前聊天窗口')
  
  // 清除当前会话的 localStorage 消息数据
  if (activeSessionId.value) {
    localStorage.removeItem(`chat_session_${activeSessionId.value}`)
    localStorage.removeItem(`chat_form_${activeSessionId.value}`)
  }
  
  // 通知 ChatAssistant 组件重新加载（会显示空的消息列表）
  // 用户发送第一条消息时才会创建真正的会话
}
```

### App.vue - onSessionInit（首次发消息回调）

```javascript
// ChatAssistant 首次发消息后回调：创建会话并添加到左侧栏
const onSessionInit = ({ localId, dbSessionId }) => {
  console.log('[App] session-init 回调:', { localId, dbSessionId })
  
  // 查找是否已存在该本地会话
  let s = sessions.value.find(s => s.id === localId)
  
  if (!s) {
    // 如果本地会话不存在，说明是首次发消息，需要创建本地会话并添加到左侧栏
    console.log('[App] 创建新会话并添加到左侧栏:', localId)
    const now = Date.now()
    s = {
      id: localId,
      title: '新对话',  // 标题会在 title-update 事件中更新为消息内容
      createdAt: now,
      updatedAt: now,
      dbSessionId: dbSessionId  // 此时已经有 dbSessionId 了
    }
    sessions.value.unshift(s)
    saveSessions()
    console.log('[App] 会话已添加到左侧栏')
  } else {
    // 如果会话已存在，更新 dbSessionId
    if (dbSessionId && !s.dbSessionId) {
      s.dbSessionId = dbSessionId
      saveSessions()
      console.log('[App] 更新会话 dbSessionId:', localId, dbSessionId)
    }
  }
  
  // 激活该会话
  activeSessionId.value = localId
  activeDbSessionId.value = dbSessionId || ''
  console.log('[App] 激活会话:', localId, 'DB ID:', dbSessionId)
}
```

### App.vue - initDbSessions（清理空会话）

```javascript
// 4.5. 清理所有空会话（豆包逻辑：左侧栏只显示有内容的会话）
const emptySessions = sessions.value.filter(s => !s.dbSessionId)
if (emptySessions.length > 0) {
  console.log(`[App] 发现 ${emptySessions.length} 个空会话，全部删除（豆包逻辑：不显示空会话）`)
  
  // 删除所有空会话及其 localStorage 数据
  for (const s of emptySessions) {
    console.log(`[App] 删除空会话: ${s.id}`)
    localStorage.removeItem(`chat_session_${s.id}`)
    localStorage.removeItem(`chat_form_${s.id}`)
  }
  
  sessions.value = sessions.value.filter(s => s.dbSessionId)
  saveSessions()
  console.log(`[App] 清理后剩余 ${sessions.value.length} 个会话`)
}
```

### Sidebar.vue - 显示会话标题

```vue
<span class="session-title">{{ s.title || '新对话' }}</span>
```

**说明**：这里显示"新对话"只是占位符，实际上按照豆包逻辑，左侧栏不应该有空会话。这个占位符只是为了防止 title 为空时显示空白。

## 与之前错误逻辑的对比

### ❌ 之前的错误逻辑

1. 点击"新建会话"会创建空会话
2. 空会话会显示在左侧栏（标题为"新对话"）
3. 系统中可能存在多个空会话
4. 空会话有本地记录但没有 dbSessionId

### ✅ 正确的豆包逻辑

1. 点击"新建对话"只是清空聊天窗口，不创建会话
2. 左侧栏只显示有内容的会话（有 dbSessionId）
3. 会话在用户发送第一条消息时才创建
4. 会话标题来自第一条消息内容
5. 不存在空会话的概念

## 用户体验

**豆包式的体验：**

1. **首次使用**：
   - 登录后看到空的左侧栏和聊天欢迎界面
   - 直接在输入框输入消息
   - 发送后，左侧栏出现第一个会话，标题为消息内容

2. **继续对话**：
   - 在当前会话中继续聊天
   - 左侧栏始终显示该会话

3. **开始新对话**：
   - 点击"新建对话"按钮
   - 聊天窗口清空
   - 输入新消息并发送
   - 在同一个会话中继续（或者可以理解为覆盖旧对话）

4. **多会话管理**：
   - 每个有内容的会话都显示在左侧栏
   - 点击切换即可查看历史对话
   - 可以删除不需要的会话

## 总结

**核心原则：左侧栏只显示有内容的会话，会话在用户发送第一条消息时创建。**

这完全符合豆包的设计理念，提供了更简洁、更直观的会话管理体验。
