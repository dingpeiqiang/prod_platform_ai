# 历史数据导入功能使用指南

## 📋 功能概述

历史数据导入功能允许你将已有的业务数据（如请假记录、销售订单等）批量导入系统，使推荐引擎能够基于这些历史数据提供智能推荐。

---

## 🚀 快速开始

### 1. 准备数据文件

在 `backend/config/import_data/` 目录下创建以下文件：

#### 数据文件（必需）
```
{formCode}.data.jsonl
```

**格式要求：**
- 每行一个完整的 JSON 对象
- 字段名必须与本体定义中的 `fieldCode` 一致
- 支持嵌套结构和数组
- 以 `#` 开头的行为注释

**示例（leave.data.jsonl）：**
```json
{"applicant_name":"张三","department":"技术部","leave_type":"年假","start_date":"2026-03-15","end_date":"2026-03-20","reason":"回老家探亲","days":5}
{"applicant_name":"李四","department":"销售部","leave_type":"事假","start_date":"2026-03-22","end_date":"2026-03-24","reason":"家里有事","days":3}
```

#### Schema 文件（可选）
```
{formCode}.schema.json
```

**用途：**
- 定义表单元数据（名称、描述）
- 配置值转换规则
- 指定用户字段

**示例（leave.schema.json）：**
```json
{
  "formCode": "leave",
  "formName": "请假申请",
  "description": "员工请假申请数据",
  "valueTransform": {
    "leave_type": {
      "annual": "年假",
      "sick": "病假",
      "personal": "事假"
    }
  },
  "userField": "applicant_name"
}
```

---

### 2. 启动系统

确保后端和前端服务已启动：

```bash
# 启动后端
cd backend
python -m uvicorn app.main:app --reload --port 8000

# 启动前端
cd frontend
npm run dev
```

---

### 3. 导入数据

#### 方式一：通过 Web 界面（推荐）

1. 打开浏览器访问前端应用
2. 点击左侧边栏底部的 **"📊 数据导入"** 按钮
3. 选择要导入的表单类型
4. 确认导入配置（可选设置限制条数）
5. 点击 **"开始导入"**
6. 查看导入结果和字段分布统计

#### 方式二：通过命令行

```bash
# 列出所有可导入的表单
python -m app.scripts.import_history --list

# 预览导入（不写入数据库）
python -m app.scripts.import_history leave --dry-run

# 导入单个表单
python -m app.scripts.import_history leave

# 限制导入条数（用于测试）
python -m app.scripts.import_history leave --limit 10

# 导入全部表单
python -m app.scripts.import_history --all
```

## 📊 导入结果说明

导入成功后，你将看到以下信息：

### 基本统计
- **源记录数**：数据文件中的总记录数
- **实际导入**：成功写入数据库的记录数
- **跳过(空数据)**：因数据为空而跳过的记录数
- **错误数**：处理失败的记录数

### 字段分布
显示每个字段的 Top 5 值及其出现次数，例如：

```
applicant_name:
  张三: 5次
  李四: 3次
  王五: 2次

leave_type:
  年假: 8次
  病假: 6次
  事假: 4次
```

### 嵌套字段处理
如果数据包含嵌套结构（如订单明细），系统会自动展平并写入 `FormHistory` 表，用于推荐引擎查询。

## 🔍 验证导入结果

### 1. 检查数据库

```sql
-- 查看导入的实例数量
SELECT COUNT(*) FROM form_instance WHERE template_id IN (
  SELECT id FROM form_template WHERE form_code = 'leave'
);

-- 查看字段历史记录
SELECT field_code, field_value, COUNT(*) as count
FROM form_history
WHERE form_instance_id IN (
  SELECT id FROM form_instance WHERE template_id IN (
    SELECT id FROM form_template WHERE form_code = 'leave'
  )
)
GROUP BY field_code, field_value
ORDER BY field_code, count DESC;
```

### 2. 测试推荐功能

在聊天窗口中尝试填写表单，系统会基于导入的历史数据提供智能推荐。

例如，输入："我要请假"，系统会推荐：
- 申请人：张三、李四、王五...
- 部门：技术部、销售部、人事部...
- 请假类型：年假、病假、事假...

## ⚠️ 注意事项

### 数据安全
- 导入过程不会覆盖已有数据，只会追加新记录
- 重复导入同一数据文件会导致数据重复
- 建议先在测试环境验证后再导入生产数据

### 性能优化
- 大量数据导入时，建议使用 `--limit` 参数分批导入
- 导入过程中数据库会有写操作，可能影响其他请求响应速度
- 建议在非高峰时段执行大规模导入

### 数据质量
- 确保字段名与本体定义完全一致（区分大小写）
- 日期格式推荐使用 `YYYY-MM-DD`
- 避免在数据中包含敏感信息（如密码、API Key）

## 🛠️ 常见问题

### Q1: 导入后推荐引擎没有反应？

**A:** 可能的原因：
1. 数据未正确导入，检查导入日志
2. 字段名与本体定义不匹配
3. 需要重启后端服务以刷新缓存

### Q2: 如何删除已导入的数据？

**A:** 目前不支持通过界面删除，可通过 SQL 手动清理：

```sql
DELETE FROM form_history 
WHERE form_instance_id IN (
  SELECT id FROM form_instance WHERE template_id = <template_id>
);

DELETE FROM form_instance WHERE template_id = <template_id>;
```

### Q3: 支持哪些数据格式？

**A:** 
- ✅ JSONL（推荐）：支持嵌套、数组、任意层级
- ⚠️ CSV（兼容模式）：仅支持扁平结构

### Q4: 导入失败怎么办？

**A:** 
1. 检查数据文件格式是否正确
2. 查看后端日志文件 `backend/logs/app.log`
3. 使用 `--dry-run` 参数预览而不写入
4. 确认数据库连接正常
