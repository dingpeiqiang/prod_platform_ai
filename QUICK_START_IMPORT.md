# 🚀 历史数据导入 - 快速开始

## 📌 功能说明

本功能提供**导入入口与引导**，让你可以轻松将已有的业务数据导入系统，使推荐引擎能够基于历史数据提供智能推荐。

---

## ⚡ 3步快速导入

### 第1步：准备数据（如果还没有）

⚠️ **重要：系统不会预置任何示例数据！**

你需要从自己的业务系统中导出真实数据，然后放入 `backend/config/import_data/` 目录。

**参考示例：**
我们在 `backend/config/import_data_examples/` 目录提供了格式参考文件：
- leave.data.jsonl - 请假数据格式示例
- sales_order.data.jsonl - 销售订单格式示例
- tariff_filing_publicity.data.jsonl - 资费备案格式示例

**但请注意：**
- ❌ 这些只是格式参考，不会被系统自动导入
- ✅ 你需要准备自己的真实业务数据
- ✅ 将你的数据文件放入 `import_data/` 目录

---

### 第2步：启动服务

```bash
# 方式1：使用启动脚本（推荐）
start-all.bat

# 方式2：分别启动
# 终端1 - 后端
cd backend
python -m uvicorn app.main:app --reload --port 8000

# 终端2 - 前端
cd frontend
npm run dev
```

等待服务启动完成，浏览器会自动打开或访问：`http://localhost:5173`

---

### 第3步：导入数据

#### 🖥️ 通过Web界面（推荐）

1. **打开导入对话框**
   - 点击左侧边栏底部的 **"📊 数据导入"** 按钮

2. **选择表单类型**
   - 从列表中选择要导入的数据类型（如"请假申请"）
   - 查看表单信息和数据预览

3. **确认导入配置**
   - 可选：设置"限制条数"用于测试
   - 阅读注意事项

4. **执行导入**
   - 点击"开始导入"
   - 等待导入完成（通常几秒到几分钟）

5. **查看结果**
   - 查看导入统计信息
   - 浏览字段分布情况
   - 点击"完成"关闭对话框

#### 💻 通过命令行

```bash
# 列出可导入的表单
cd backend
python -m app.scripts.import_history --list

# 预览导入（不写入数据库）
python -m app.scripts.import_history leave --dry-run

# 执行导入
python -m app.scripts.import_history leave

# 限制条数（测试用）
python -m app.scripts.import_history leave --limit 10
```

## ✅ 验证导入成功

### 方法1：查看导入结果

导入完成后，界面会显示：
- ✅ 导入成功提示
- 📊 统计数据（源记录数、实际导入数等）
- 🔍 字段分布（Top 5值及出现次数）

### 方法2：测试推荐功能

1. 在聊天窗口输入："我要请假"
2. 系统会自动识别意图并生成请假表单
3. 填写字段时，会看到基于历史数据的智能推荐

例如：
- **申请人**：推荐张三、李四、王五...
- **部门**：推荐技术部、销售部、人事部...
- **请假类型**：推荐年假、病假、事假...

## ❓ 常见问题

### Q: 导入后没有看到推荐？
**A:** 
1. 确认导入成功（查看导入结果）
2. 刷新页面重新生成表单
3. 检查字段名是否与本体定义一致

### Q: 可以重复导入吗？
**A:** 可以，但会导致数据重复。建议先清空再导入，或使用不同的数据文件。

### Q: 如何删除已导入的数据？
**A:** 目前需要通过SQL手动删除，详见[详细说明文档](./历史数据导入功能说明.md)。

### Q: 支持哪些数据格式？
**A:** 
- ✅ JSONL（推荐）：每行一个JSON对象，支持嵌套
- ⚠️ CSV（兼容）：仅支持扁平结构
