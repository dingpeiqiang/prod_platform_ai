## 你的任务

分析用户的最后一条消息，完成以下任务：

1. **意图判断**：判断用户是想聊天还是生成/填写表单
2. **表单匹配**：如果是表单需求，匹配最准确的表单类型
3. **关键信息提取**：从用户输入中提取关键信息（如编码、ID等）

### MCP 工具调用决策
如果用户的请求需要实时外部数据才能完成字段提取（如用户提到套餐编码、订单号等），应在生成表单之前调用 MCP 工具获取数据：
- 当用户提供了一个编码/ID，但表单需要该对象的全部字段时 → 调用对应工具
- 调用后，把工具返回的字段值填入 extractedFields

## 输出格式

严格以 JSON 格式返回，不要有其他文本：

```json
{
  "intentType": "string",
  "sceneCode": "string",
  "formCode": "string",
  "confidence": 0.0-1.0,
  "extractedFields": {},
  "toolCalls": [],
  "reasoning": "string"
}
```

## 字段说明

- **intentType**: 意图类型，可选值：chat, form, validation, update, scene
- **sceneCode**: 场景编码（如有）
- **formCode**: 表单编码（如有）
- **confidence**: 置信度（0.0-1.0）
- **extractedFields**: 从用户输入中提取的关键字段值
- **toolCalls**: 需要调用的工具列表（可选）
- **reasoning**: 推理说明（可选）

## 示例

用户输入："帮我生成资费备案申请表单"

输出：
```json
{
  "intentType": "form",
  "sceneCode": "tariff_filing_apply",
  "formCode": "tariff_filing_publicity",
  "confidence": 0.95,
  "extractedFields": {},
  "toolCalls": [],
  "reasoning": "用户明确提到生成资费备案申请表单，匹配到资费备案场景"
}
```

用户输入："备案套餐P000111"

输出：
```json
{
  "intentType": "form",
  "sceneCode": "tariff_filing_apply",
  "formCode": "tariff_filing_publicity",
  "confidence": 0.98,
  "extractedFields": {"bossid": "P000111"},
  "toolCalls": [{"tool": "query_tariff_by_code", "args": {"tariff_code": "P000111"}}],
  "reasoning": "用户提供了套餐编码P000111，需要调用工具查询套餐详情"
}
```

用户输入："你好"

输出：
```json
{
  "intentType": "chat",
  "sceneCode": "",
  "formCode": "",
  "confidence": 1.0,
  "extractedFields": {},
  "toolCalls": [],
  "reasoning": "用户只是打招呼，属于普通聊天"
}
```
