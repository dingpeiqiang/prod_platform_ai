根据以上用户输入，结合场景层级结构，精准识别出具体的三级场景编码。

## 场景匹配流程

**第一步：分析用户输入**
- 提取用户输入中的业务关键词

**第二步：缩小搜索范围**
- 根据关键词确定候选的中心域和业务域
- 组合中心域+业务域形成搜索范围

**第三步：匹配具体场景**
- 在确定的范围内查找匹配的场景
- 如果场景描述模糊，结合中心域和业务域综合评估

**第四步：返回结果**
- 匹配到场景 → 返回场景编码
- 未匹配到场景 → 返回空字符串

## 输出格式

**只输出JSON，不要添加任何额外文本！**

```json
{"sceneCode":"string","confidence":0.0-1.0,"reasoning":"string"}
```

## 字段说明

- **sceneCode**: 匹配到的三级场景编码，未匹配返回空字符串
- **confidence**: 置信度(0.0-1.0)
- **reasoning**: 推理说明

## 匹配策略

1. 场景优先：直接匹配场景级关键词优先
2. 层级辅助：用中心域+业务域缩小范围
3. 组合评估：结合层级综合判断
4. 无场景返回空字符串

## 重要规则

1. **只返回JSON**：不要添加"用户输入"、"输出"等额外文本
2. **场景优先**：最终目标是匹配三级场景
3. **精确匹配**：场景编码必须完全匹配配置值
4. **无场景返回空**：找不到场景时sceneCode返回空字符串

## 示例

正确输出：
```json
{"sceneCode":"tariff_filing_apply","confidence":0.95,"reasoning":"用户提到资费备案，匹配到资费备案申请场景"}
```

无匹配输出：
```json
{"sceneCode":"","confidence":0.3,"reasoning":"未匹配到相关场景"}
```
