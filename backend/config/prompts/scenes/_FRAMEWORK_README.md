# 场景动作提示词框架

## 概述

场景动作提示词（Scene Action Prompt）是连接场景识别与能力调度的核心。它定义了大模型在识别到特定场景后的思考和执行方式。

## 目录结构

```
config/prompts/scenes/
├── _FRAMEWORK_README.md           # 本文档
├── _templates/                    # 模板目录
│   ├── form_generation.txt        # 标准表单生成模板
│   └── form_with_mcp.txt          # 带工具调用的表单模板
└── [场景编码]_prompt.txt         # 具体场景的提示词文件
```

## 提示词变量

在提示词中可以使用以下变量：
- `{scene_code}`: 场景编码
- `{scene_name}`: 场景名称
- `{form_code}`: 关联表单编码
- `{current_date}`: 当前日期
- `{current_time}`: 当前时间

## 开发指南

### 1. 创建新场景提示词

1. 复制合适的模板文件
2. 重命名为 `[场景编码]_prompt.txt`
3. 根据场景需求修改内容
4. 在场景配置中关联该提示词文件

### 2. 提示词结构建议

标准场景提示词应包含以下部分：
- **角色定义**: 明确助手的定位和职责
- **场景概述**: 简要描述场景功能
- **核心目标**: 场景要完成的关键任务
- **处理流程**: 分步骤说明执行逻辑
- **注意事项**: 边界情况和特殊处理
- **输出要求**: 明确返回格式

### 3. 最佳实践

- **清晰具体**: 避免模糊描述，给出明确指令
- **容错设计**: 考虑各种异常情况的处理
- **用户友好**: 交互自然流畅，让用户理解操作
- **格式规范**: 严格按约定格式输出结果
